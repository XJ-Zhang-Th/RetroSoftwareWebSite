一起来抓网虫

　　在上海见到了歪歪，闹出了不少的笑话。至今想来，仍然让人忍俊不禁。可惜的是，写多了大有骗稿费之嫌，只能以后再聊了。另外，三公子最近发现了一个很好的关于中国共享软件的网站（http://cnshare.yeah.net),上面发布了从DOS时代到网络时代的优秀的共享软件。该网站得到了国内几乎所有写共享软件的作者的支持。喜欢共享软件的朋友去看看吧。好了，废话不说，还是来看看我抓住的“网虫”吧。
　　看了这期标题，一定觉得奇怪，网虫？咬了网页制作软件的虫子！没错，就是Frontpage 2000，这次轮到它了。如果你在用Frontpage 2000制作网页时遇到了问题，千万不要惊奇，微软的东西向来在捉虫手记里占有很大的分量，这次更是想不到，收集了一下最近的资料，居然大部分都是关于Frontpage 2000这个玩意的。大家一起来看看吧。

不是服务器扩展的错

　　在FrontPage 2000中列出整个网站时，你可能会收到一个错误信息提示“the server extensions are not installed（没有安装服务器扩展）”。微软说这个错误只有当你在Apache 1.2.6 Server上使用了Change_server.sh补丁后才会产生。解决方法是从http：//www.apache.org上下载1.3.3版本。
　　另一个方法是：
　　1) 删除/Usr/Local/Frontpage/Version4.0/Apache－fp/Httpd
　　2) 将/usr/local/frontpage/version4.0/apache－fp/httpd 复制
到 /Usr/Local/Frontpage/Version4.0/Apache－fp/Httpd
　　小知识：
　　FrontPage服务器扩展：是Web服务器中的一组程序，它扩展了FrontPage的功能。FrontPage服务器扩展的功能包括：Web站点的创作；Web站点的管理；服务器一侧的激活和Web页面的改善。

路径的问题

　　当你在IIS(Internet Information Server) 4.0上安装微软的FrontPage 98或者FrontPage 2000 服务器扩展的时候，可能会遇到以下错误信息："Unable to read configuration information for Microsoft Internet Information Server.（不能读微软的互联网信息服务器的配置信息）" 当你需要列出整个Web时，也会遇到这样的提示：“the server could not complete your request（服务器不能完成你的要求）”。 如果按下“详细情况”按钮可以得到以下信息：“The specified module could not be found（找不到指定模块）" 或者 “Cannot create folder （不能建立文件夹）c:[folder〗 ―vti_pvt."注意，这儿的路径后面有一个空格。 微软说这种情况是在服务器指向了一个非法路径，或者路径最后包含了一个空格的路径时发生。
　　修正该错误的方法很简单，检查路径，并且去掉其中的空格。

搜索也出错

　　当微软在Hway.net这个世界上最大的FrontPage Web 服务器提供商上进行的首次FrontPage 2000 服务器扩展的展出时发现了一个新的错误。这个错误是在用户用浏览器试图从FrontPage创建一个搜索时发生，用户获得的不是一张包含搜索结果的网页，而是一个无法找到http:///_vti_bin/shtml.exe/的错误信息。
　　Hway.net指出，使用FrontPage 主题的网页这个情况会更加严重，但该问题在不使用FrontPage主题的网站也会发生。
　　Hway.net的提供修补方法是：
　　重新计算超级连接，如果仍然有错误，重新安装FrontPage 2000 服务器扩展，如果还没有用，再次安装FrontPage 2000 服务器扩展。
　　很不幸的是，这个“解决方法”破坏了FrontPage的其他部分。重新安装FrontPage 2000 服务器扩展会破坏已经存在的网站结构，同时会将所有的子站点移到根目录上。而且这个动作还会破坏子站点的所有用户名和密码。
　　其实修复这些错误，你需要：打开根目录，创建一个新的FrontPage网站，使用以前的子站点相同的名字（在FrontPage Explorer中作为一个文件夹形式显示的部分），这个操作可以自动将在根目录下的目录备份到一个分离的子网站。
　　“我们为FrontPage 2000 服务器扩展做了很多的Beta测试”Hway.net说，“但是在测试版本中从来没有遇到过这种情况。”
　　微软拒绝承认这是FrontPage 2000的一个Bug，他们提出这是由Hway.net在制作过程中的一个错误导致的问题。不过，即使这样，微软仍然会考虑提供一个解决方案。“我们目前正在评估是否要为我们类似Hway.net这样的合作伙伴们提供一个简单的脚本来对所有网站的超级连接进行重新计算，还是专门提供一个补丁。”

写不完的IE5

　　微软又为IE5出补丁了。这次的问题出在IE5 for Windows 95/98上，对NT没有影响。这个被称为“Malformed Favorites Icon”的Bug允许一些无聊却可恶的网管悄悄启动你系统上的程序而不被你发觉。我们知道，在IE 5的收藏夹里的所有项目图标都是可以单独定义的，甚至可以设置为服务器上提供的图标。但是，这个功能却能够触发一次缓存溢出，这个时候该Bug提供了在你的系统激活某些程序的可能。要修补这个漏洞，可以到http://www.microsoft.com/windows/ie/security/favorites.asp去找出相关补丁。
　　一个在早期IE版本中使用过的Active X控件，在IE 4/5中已经失去作用，却仍然被安置在程序中。这个东西允许网管去读取你硬盘上的内容。要修复该漏洞，IE4和IE5的用户可以在 www.microsoft.com/windows/ie/security/favorites.asp下载一个补丁。
