在“老奔”上使用大容量硬盘

　　如今这时代，没有人会嫌硬盘大，就像住房一样。前不久，经过仔细比较，又参考了多家广告的宣传，怀着对IBM品牌无比信任的心情，终于弄到一款号称IBM桌面之星的13.5GB大硬盘。当时也没有多想什么8.4GB的限制，只是想大硬盘卖得这么火，不会有买回去做摆设的，没想到请回了这么个大硬盘，却花费了好几天时间，外带N个小时的网上漫游才搞定。真是要想得空间，就得花时间！
　　首先，把它装在我那IBM 133 的老奔上，折腾多时，尝试过DM、NORTON磁盘医生、PQMAGIC、GHOST等凡是与硬盘有关的分区和格式化软件，系统还是只认8.4GB；换上IBM PⅡ机器，情况依旧；这之后又换过了许多机器，只是有一款“和光”PⅡ 机器，与它有点缘分，识得它的真面目；有一发烧初哥告之，在这台“和光”机器上分区、格式化，再拿到其它机器上就可以使用了，本着“死马当活马医”的原则，照此方法做，还是无结果。
　　然后，与销售厂家联系、上Internet各BBS 站点请教，得到了许多朋友的热心帮助，也得到了许多答案，其中最多的建议是升级机器主板的BIOS，使之支持大于8.4GB的硬盘。但使用的机器多种多样，并不是每个机器都能简单地进行升级的，再说升级失败了怎么办，CIH没能破坏的BIOS被我破坏了，岂不可惜？
　　带着还有5.1GB的空间无法使用的焦急心情，日子一天天的在过去，眼看着硬盘的价格在下降，我的火气却在上升。直到一个周末的深夜，我对着这款13.5GB的硬盘出神，忽然在这个盘体上发现了网址www.ibm.com／Harddrive，谢天谢地，IBM就是IBM，你看人家大公司考虑得多周到。但是你别高兴得太早，IBM真是一个大的“官僚”公司，虽然告诉了你一个网址，但如果你不在它的迷宫里转个天昏地暗，三进三出，然后下载N个PDF文件，仔细学习完，要想取得真经，真是梦想。就这样从这个IBM子站到另一个IBM子站，转了二三个小时，总算找到了一个名为IBM－DM.EXE的可执行文件，长度为1037760字节。下面具体介绍设置的步骤：
　　1．在WIN95／98下执行IBM－DM.EXE文件后，产生一张称为“Disk Manager Installation” 的系统盘，再制作一张WIN95或WIN98的系统盘，包含Format、Fdisk等文件，供引导系统用，再正常关机。
　　2．为了安全起见，把其它硬盘去掉，只留下这个IBM大硬盘，开机。进入CMOS设置，把这个硬盘的类型设为AUTO，引导顺序设为“A：C：”，保存设置后启动（软驱中放入“Disk Manager Installation”系统盘）。
　　3．屏幕显示“Ontrack Standalone Operating System V1.00.09a”，按下回车键，显示欢迎信息，再回车，出现“Disk Manager Main Menu”，下有三个选项“（E）asy Disk Installation”、“（A）dvanced Options”、“（V）iew／Print Online Manual”：
　　选择“Easy Installation”，回车后，检测到一个硬盘，型号为IBM－DTTA－351350，选择操作系统（WIN95／WIN98／WIN NT等），再选择硬盘分区表的格式。若选FAT32，按ALT＋C键，显示硬盘总容量为13522.3MB，只能格式化为一个分区，确定后，约十几秒钟，格式化完成，根据提示进行第4步。若选FAT16，按ALT＋C键，显示总容量为13522.3MB，可格式化为七个分区：其中前6个分区容量各为1932.9MB，后面1个分区的容量为1924.7 MB，确定后，约十几秒钟，格式化完成，根据提示进行第4步。
　　先选择“Advanced Option”，再选择“Advanced Disk Installation”，列出硬盘数量及型号，选择操作系统及分区表格式，有FAT32和FAT16供选择，与上面不同的是，可以自己定义分区的数量和大小，格式化完成后，根据提示进行第4步。
　　4．生成“Ontrack Boot Diskette for ” 磁盘，然后用该盘启动机器，用“SYS C：”传送系统到C盘，取出软盘。
　　5．热启动，在CMOS中改为从硬盘启动。
　　6．启动时，根据提示按ESC键，第一次启动完成。
　　以后每次启动时，都要进行第六步。
　　小结：
　　1．“Ontrack Boot Diskette for ” 软盘一定要保管好，以后做硬盘时要用，否则“再生”会很麻烦。
　　2．在上述第3步产生分区时，所有的分区表格式要么是FAT32格式，要么是FAT16格式，而不能两者混用。如果想C盘采用FAT16格式，与DOS兼容，其它盘采用FAT32格式以节省空间，可以选择第3步第2方法中的FAT16分区，将硬盘分成6个分区（容量为2047MB）＋1个分区（容量为612MB），进行第4～6步后，用WIN98下的FDISK命令，把第1分区（容量为2047MB）的FAT16保留，其它的Logical drive全部删除，再重新生成大于2047MB的FAT32分区，然后再格式化，即可实现FAT16和FAT32混用。欢迎来Email交流，地址：Chenmoon@163.net。
（ 南京　陈明）
