我给公司建了一个Intranet

　　我公司是一家以生产面粉为主的外资企业，公司拥有电脑40多台,主要用于人事行政、财务会计等管理部门的办公以及生产自动控制。
　　笔者作为一名财务人员，也是一名电脑爱好者，一直致力于公司的财会电算化等应用工作，对编程、网络及Internet也有一定了解。基于此，公司领导“下令”我用最少的Money搞出一个类似Internet的信息网，我打下包票：“这不就是个Intranet吗？三个月搞掂！”说归说，可也不敢大意，经过规划、购买、安装、调试、推行等一系列实施过程，此信息网络终于成功运行。
　　对于Intranet技术实现方法，《电脑报》及其他计算机报刊上均有不同程度的介绍，只不过缺少“整合”而已，尤其是在企事业单位的实际应用。故本文偏重在方案的设计及应用方面，辅以一些注意事项及要点，如需详细了解，可随时与我联系,共同进行探讨。

一、方案设计

　　A.Intranet提供的服务
　　鹏泰信息网(PangThai Intranet)以WWW、FTP、E－mail等方式为信息网中的每台计算机提供下列服务：
　　1．公司资料共享
　　共享公司的各种信息资料，发布公司内部刊物的电子版。
　　2．实时信息传递
　　传递有关粮食体制改革、市场状况等粮油方面的动态信息；转载、摘编国内、国际重大新闻信息。
　　3．动态数据查询
　　动态查询产品的生产、销售、库存等实时数据；动态查询客户、员工的档案信息。
　　4．报表服务
　　提供销售、生产、会计、统计等报表供相关人员查阅、分析。
　　5．网络教室及软件下载
　　6．联入Internet
　　通过代理服务器使公司的计算机均能以低廉费用接入Internet。
　　7. 其他信息服务
　　开通部门之间、员工之间的E－mail服务；开设留言簿、聊天室、电子论坛等栏目以增强信息网的趣味性、娱乐性。
　　B.硬件规划：
　　1.拓扑结构
　　采用100Base－TX高速以太网。考虑管理部门集中在办公楼的底三层、而财务部已有服务器进行账务及其他专用处理、工厂离办公楼较远等因素，故采用分层星型和总线型的混合拓扑结构。
　　2.硬件设备
　　各工作站之PC机、Modem及财会部之文件服务器、网络打印机、集线器均为原来的设备，故只需投资购买如下硬件设备：
　　Web Server: 联想万全2400服务器(PⅡ400/2 双CPU、256M内存、9G热插拔硬盘)
　　Switch HUB: D－Link的DES－1016 10/100M 16口交换器
　　HUB：D－Link的DFE－916X 10/100M 16口集线器
　　电缆：5类非屏蔽双绞线若干；粗缆若干
　　其他：RJ45头、 收发器、AUI TO UTP 、信息插座/面板、接线工具等
　　C．软件规划
　　1.服务器端
　　a.操作系统：信息网之Web Server采用Windows NT Server 4.0 中文版；财会电算化之File Server采用Netware4.11。
　　如果预算宽裕,则可在Web Server使用Microsoft BackOffice Small Business Server 4.0。
　　b.WWW及FTP服务器：采用NT4.0附带的IIS2.0 ＋ SP3 ＋ ASP
　　c.E－mail服务器： Exchange5.5评估版
　　d.代理服务器： Sygate2.0 或Wingate2.1
　　e.网页制作： Frontpage 98 ＋ Image Composer 1.5 ＋ Ulead COOL 3D 2.0
　　f.数据库： 考虑易用性及安全性暂采用Access 97中文版＋32位ODBC (建议采用SQL6.5)
　　g.压缩软件：WinZip 7.0，使FTP下载资源均采用＊.Zip压缩格式
　　2.客户端
　　a.操作系统： Win95中文版 或Win98中文版
　　b.浏览器： 全部采用IE 4.01中文版以支持ASP
　　c.Office 97中文版：报表采用Excel 97格式，文档采用Word 97格式；数据库采用Access 97格式。

二、方案实施

　　A．硬件部分
　　硬件部分的实施主要是根据设计方案购买硬件设备及按照拓扑结构进行网络布线,此处不再赘述。
　　需注意的是:
　　1.服务器除稳定性外重点考虑的是硬盘速度和内存大小,硬盘应选用适合并发数据请求的SCSI接口；内存则应尽可能大；
　　2.布线过程中制作RJ45头时，不能简单地一一对应连接，应将Ethernet使用的1、2、3、6脚采用相互缠绕的两对线, 以防止串扰；
　　3．双绞线的最大距离为100米，利用有源HUB的级联功能也能扩展至200～300米，进行较长距离的连接时建议采用粗缆(500米)。
　　4．布线时应进行综合布线，以便在办公室调整和增加计算机时均能方便接入而不需重复布线。

三、方案应用

　　A 应用规划
　　1.调查信息需求
　　通过下发调查表和召开座谈会等形式了解各部门的工作内容、业务流程，让业务人员讨论并确定其信息需求。
　　2.规划网页结构
　　Intranet的整个Web页面基本上采用目录与正文的框架形式，即左面为栏目索引，右面为实际内容。共计10个大栏目，每个栏目下又可细分多个小栏目，详述如下：
　　公告栏发布各种文件、通知、告示等办公信息以及其他栏目的更新索引等
　　公司简介
　　a.公司介绍 b.组织结构 c.历年大事
　　d.制度汇编 e.员工状况 f.图片展览
　　《鹏泰通讯》
　　a.《鹏泰通讯》 b.《工会园地》
　　c.《粮油信息》 d.国内新闻 e.国际新闻
　　产品介绍
　　a.产品简介 b.技术指标 c.技术服务
　　d.产品价格 e.产品查询
　　销售动态
　　a.营销政策 b.销售统计 c.市场传真
　　d.信息反馈 e.售后服务
　　财务报告
　　a.生产日报 b.销售日报 c.会计报表 d.统计报表 e.报表查询
　　信息服务
　　a.网络书屋 b.E－mail服务 c.留言簿
　　d.鹏泰论坛 e.实时聊天室 f.个人主页
　　网络教室
　　a.MS－DOS b.Windows c.Office
　　d.Network e.Internet f.Tools
　　g.Others
　　软件下载
　　a.公司资料 b.汉字系统 c.办公软件
　　d.程序设计 e.驱动程序 f.实用工具
　　g.病毒防杀 h.其他
　　热线站点
　　常见的、最新的、最好的Internet站点介绍。
　　3.数据库的动态发布：
　　考虑用CGI访问数据库的开发难度，决定采用ASP(Active Server Page)技术来实现数据库的动态发布。而ASP又可以通过三种方法访问数据库，即IDC(Internet Database Connector)、 ADO(Active Data Objects)、RDS(Remote Data Service)，通过对比决定使用功能强大、方便灵活的ADO方式。
　　4．应用推广
　　a.应用培训――开办网络快训班，对信息网进行简单介绍，让办公人员熟悉其查找、获取信息的方法，并培训其利用FrontPage98发布、更新相关信息；
　　b.信息传递――将部门间相互传递的文件、报表等信息由原来的 制作―>打印―>复印―>分发―>接收 发布方式 改为:制作―>(保密)―>E－mail分发―>接收―>打印(可选) ―>(公开)―>WEB发布―>浏览―>打印(可选)
　　c.动态数据――动态发布的数据库均统一放置在服务器上，并由各维护人员按其权限随时登录至服务器进行修改、更新；
　　d.电子刊物――将《鹏泰通讯》、《工会园地》、《粮油信息》等内部刊物改为电子版，先并行后逐步取消其书面发行方式。

四、信息网维护

　　A．更新
　　WEB页面的版面、样式更新；公司共享资料更新；网络教室、软件下载内容的更新；聊天室及电子论坛的维护等工作由管理员进行；
　　各种报表、数据库的更新以及信息的发布由相关部门的计算机应用人员依据相关权限随时从本地机登录至服务器上。
　　B．备份
　　由于考虑最低投资，未采用磁带机备份，而使用磁盘镜像技术容错，这样不但得到完整的数据备份，而且还提高了系统的性能。
　　利用NT提供的AT命令每日将重要数据自动备份至MO光盘机。
　　定期对重要数据和资料进行归类、汇总、存档，并转存为光盘片,以建立电子档案。
　　C．诊断
　　由于该信息网主要采用TCP/IP协议,在网络的调试中主要采用了以下几个诊断程序:ping、ipconfig、winipcfg、nbtstat、netstat、hostname等。
