彻底删除Office2000

　　Office2000已经推出一个多月了，我试用之后觉得不合我意，于是全部卸载了它。卸载后，一查注册表，发现还有许多的垃圾信息存于其中。手工删除却太麻烦。于是我上网，直奔Office2000的HomePage，发现了Office2000的卸载工具――Office 2000 File and Registry Eraser Utility（Office2000文件和注册项删除工具）。
　　Office 2000 File and Registry Eraser Utility是由微软公司编写的，但是微软公司并不对其提供技术担保。文件大小为192KB，文件名为Eraser2k，是一个自解压文件。国内有许多站点收录有该软件。关于详细说明如何删除Office2000，可以去http:／／support.microsoft.com／support／kb／articles／q219／4／23.asp探究。注意：该程序不能运行于Windows2000环境。
　　运行Eraser2k.exe，出现“许可协议”对话框，之后是选择存放解压文件的路径。共有四个文件：Eraser2000.bat、offcln9.exe、Offcln9.opc和Readme.txt。
　　彻底删除Office2000步骤如下：
　　1．要彻底删除Office2000在注册表中的信息，必须先运行Office2000的安装程序，选择“全部删除”，将Office2000卸载。这一步一定要先做，而不能直接运行Eraser2000.bat。
　　2．运行解压出来的Eraser2000.bat文件，出现Microsoft Office删除向导，（如图）
　　3．选择“Yes”，它将会提示你“You may need to reinstall certain applications after using this option because some shared files and settings may be deleted（这一操作将删除一些共享文件和设置，可能需要重装某些应用程序）” ，选择“Yes”开始对系统扫描并删除文件，然后要求重新启动系统。重新启动，注册表中的许多Office2000的垃圾信息和一些残余文件已被清除干净。要查看卸载记录，可以到Offcln9.exe所在目录下打开Offcln9.log查看。

　　意外事件的处理：
　　1．删除了共享文件
　　一旦Offcln9.exe删除了部分共享的文件，将导致部分程序无法运行，解决的办法是重新安装一遍该程序。
　　2．删除了Office的字体
　　Offcln9.exe在删除Office2000时会删除掉其安装时装入的字体，如Times New Roman 、 Tahoma等，这将使你的部分作品文档显示乱码令你难以确认。解决的办法也是重新安装Office2000。
　　3． 删除了Rich Edit Control
　　如果Offcln9.exe删除了Rich Edit Control，将会导致一些程序如WordPad运行错误并出现如下提示：Could not load the RichEdit control DLL。解决的办法是从Office CD－ROM Disk3中将Riched20.dll 拷到Windows\System 文件夹。
　　4． 运行Office Maintenance Mode Setup错误
　　打开“控制面板”中的“添加／删除”程序，无法运行Office安装程序，并出现如下提示：“An error occurred while trying to remove Microsoft Office 2000 Premium。Uninstallation has been cancelled”。出现这一问题的原因是没有先运行Office 2000的安装程序进行删除，而是先运行了Office 2000 File and Registry Eraser Utility程序。解决这一问题的方法是先运行Office2000的安装程序，选择“全部删除”。
　　Office 2000 File and Registry Eraser Utility是为删除Office 2000卸载后的残余信息而开发的补充程序，并不能代替Office 2000的安装／删除程序。这一点请在应用时务必注意，而且一定要先备份注册表文件，以防万一！
