共享资源管理器评测报告

　　任何一种操作系统，都是以文件作为其基本单位，这必然会涉及到文件的执行、复制、移动、删除等。同时，一个操作系统所涉及到的多种资源（如：多媒体功能、网络功能及FTP功能）等多种操作，就需要一种对其进行管理的工具――资源管理器。Windows 9X作为一种可视化的、普及率最高的操作系统，它通过内置的“Windows 资源管理器”来进行文件管理。
　　无论最先使用Microsoft Windows 3.X的用户，还是首先接触Microsoft Windows 9X的用户，一定都对其内置的资源管理器――“Windows 资源管理器”比较熟悉。但从Internet开始普及后，用户对Windows内部的资源管理器就已经不感兴趣了，认为它的功能实在是太“弱小”了，于是程序高手们按捺不住自己的想法，着手设计自己理想的资源管理器，通过Internet让其传到世界各地。现在网上流行有许多资源管理器，它们无论从界面，还是功能上都有很大差异。所以，为了给读者朋友在选择资源管理器时作一些参考，电脑报评测室这次共收集了近20个资源管理器，从其中选择了七个具有代表性的资源管理器进行评测。
　　在这次评测过程中，我们使用的系统配置为：赛扬300A、64M内存PC100、S3 Savage 3D显卡、昆腾火球七代6.4G硬盘、128Kbps专线连互联网。操作系统：简体中文Windows 98。

资源管理器的选择

　　为了准备这次的评测，我们小组的成员提前一个月就开始到网上跌打滚爬，终于不负有心人，本次收获颇丰，共搜索到近20个资源管理器。为了从其中选择出具有代表性的资源管理器，特制定了以下五个标准：1.资源占用率不高（不浪费太多资源）；2.基本功能（执行、复制、移动和删除）操作简单；3.对文件压缩功能的支持；4.对FTP传输功能的支持或有自己的独特功能；5.性价比。
　　经过小组成员的几个不眠之夜的努力工作，按照以上标准进行测试，发现这些资源管理器都有其强大功能，它们并不像当初的文件管理器那样单纯地对文件进行基本的操作，现已扩展到文件压缩、文件预览、浏览图形文件以及FTP文件传输等多种功能。最终，我们确定了以下七个具有代表性的资源管理器：Windows Commander 4.0 Beta 2(我们使用的是内部测试版，目前网上只提供3.53版下载)、Turbo Browser 98 6.11、FileProbe 1.01.01、Drag and File Gold 4.5 Beta 9、Explor2000 1.21.1、EF Commander 2.32 Build、WinNavigator 1.50。

资源管理器的安装

　　这次选出的七个资源管理器都属于共享软件，可以从其网址自由下载。在使用一定时间后必须缴费注册，并且这七个资源管理器无一例外地都支持Windows 9X及Windows NT操作平台。
　　下载文件的大小除FileProbe较大（5917KB）之外，其余六个资源管理器都在1MB左右。七个资源管理器都需要进行安装才能执行，其中，Windows Commander 4.0 Beta 2、EF Commander 2.32 Build、WinNavigator 1.50、FileProbe 1.01.01和Turbo Browser 98 6.11必须解压缩后才能进行安装（如图1为解压缩FileProbe），Drag and File、Explor 2000属于自解压安装，不需要先解压。它们的安装过程都非常简单，只要指定需安装的目录即可。七个资源管理器经过安装后，都会在“开始”菜单中建立快捷图标。但只有Drag and File 、Explor2000、FileProbe和Turbo Browser 98支持在Windows 9X系统中卸载。

　　能否支持中文，对许多用户来说是非常关键的，毕竟中文是我国的母语及官方语言。但这些资源管理器中支持中文的只有Windows Commander 4.0，它还支持英、德、法、意大利、丹麦、瑞典、挪威、荷兰、西班牙、俄、匈牙利、希腊、土耳其等多种语言。而Drag and File、FileProbe和EF Commander 也只支持英文一种版本，其它的一般都支持三种语言以上。在我们参选的七个资源管理器中只有Windows Commander 4.0支持中文。

试运行

　　为了准确测试这七个资源管理器的启动运行时间，我们小组专门用VC写了一个检测运行时间的小程序。在我们评测所使用的系统上经过近10次的评测，最终获得的评测结果是：第一次启动时间有比较大的差异，Windows Commander 4.0、WinNavigator和EF Commander 的运行时间较短，只用了2秒，而Drag and File 和FileProbe相对较长，分别为6秒和5秒。第二次启动（退出资源管理器，而未重启动系统）除WinNavigator 和EF Commander 未发生变化外，其它5 个资源管理器所用时间都相对减少。由此看来，这5个资源管理器充分利用了Windows 的Catch功能，优化了自己的启动过程。
　　资源的占用率也是评测一个软件的重要依据。我们对这七个资源管理器在CPU占用、内存占用及GDI资源占用也分别进行了多达10次的测试（使用具有权威性的测试软件――TaskInfo98进行评测，如图2）。经过综合评比，选出了资源占用率低的第一名：Windows Commander 4.0，以0.05％的CPU占用率、1.56MB的内存占用以及2％的GDI占用率夺魁。

　　
　　　　　　　　　　　　　图2

　　我们软件评测小组成员一致认为：可操作性是判别一套软件实用价值的重要因素。可操作性并不在于其有无漂亮的外观，而在于它是否支持Windows 标准风格，使用户能在从未使用过该软件时就会很快上手。对于这七个资源管理器而言，操作界面各具特色：Windows Command 4.0 操作界面不算华丽，它分左右两个独立的工作窗口（也可以设置为与Windows资源管理器相似），在其上面可以通过驱动器工具按纽选择当前的驱动器，方便操作，并且在最下方还设置有七个快捷按钮：从F3到F8及Alt＋F4，分别表示：观察、编辑、复制、移动、新建目录、删除和结束等功能（如图3）。由于它使用两个独立的工作窗口，而且目录结构属于单目录结构，与Windows资源管理器的多目录结构不一样（虽然有两个窗体，但左边一个是目录区，右边是文件区），但它对于复制、移动及新建目录等操作非常方便。WinNavigator、EF 的界面非常相似。在Windows Commander 4.0中还可以添加外挂程序，特别对未使用Windows 98并且在Windows95中未安装IE4的用户非常实用，有的用户直接将Windows Commander 4.0 作为桌面进行操作。

　　
　　　　　　　　　　　　图3

　　Explor 2000号称“无菜单的资源管理器”，在其界面上看不到菜单的踪影，但只要点击其上面的一些工具按钮就会发现并非如此，它只是将传统菜单以可视化菜单方式显示。Drag and File 资源管理器有着与Windows资源管理器相似和非常友好的界面，但它的功能又比Windows 资源管理器强大，易操作。它也分为上下两个独立的工作目录窗体，各自又属于多目录结构。它还有一个特点就是在菜单中使用“Window”→“New Window”可以共打开12个相同的窗体，并且按出现的个数来安排每个窗体的大小，每次出现新的窗体后，大小都要作相应的调整，以尽量达到平均分配。每个窗体之间支持相互拖放功能，对于一个文件需要复制到多个目录时非常有效。通过窗体菜单“Close All DF Windows”可以全部关闭通过“New Window”生成的窗体。

资源管理器基本功能测试

　　为了对这七个资源管理器的基本功能（包括复制、移动、删除和新建目录等）进行测试，并且比较它们的速度差异，我们特地准备了512个文件（共82个目录、216MB）进行测试。Turbo Browser 98和 Explor2000没有自己内置的复制、移动和 删 除模 块。 它们只是调用Windows标准动态链接库Shell32.dll中SHFileOperation函数进行操作。其余的五个资源管理器都带有自己的复制和移动操作的功能（如图4 )，它们相对于Windows 资源管理器而言，复制和移动速度都比较快。FileProbe无论是复制、移动，还是删除的速度都是第一名，它复制时间只用了1分28秒，移动时间1分30秒，删除时间4秒，而因为Turbo Browser 98和 Explor2000因调用Windows标准动态链接库Shell32dll的ShfileOperation函数所用的时间最长：复制时间2分14秒、移动时间2分、删除时间5秒，虽然它们的删除功能比其它四个资源管理器（FileProbe除外）所花时间都短。

　　
　　　　　　　　　　　　　　图4

　　在我们的测试中，发现除Windows Commander 4.0、Drag and File 、FileProbe、Win Navigator和EF Commander这四个资源管理器都支持两处种复制及移动操作（资源管理器自带及调用Windows 的标准动态库Shell32.dll），而Turbo Browser 98和Explor2000就属于第二种。这七个资源管理器都支持文件拖放功能，鼠标右键帮助菜单功能，属于Windows 风格（实际上这种功能也是调用Windows 动态库）。
　　由于Windows 9X是属于多任务、多线程的操作平台，所以，我们也希望资源管理器也能支持多线程复制、移动等操作。正好Windows 资源管理器的复制和移动功能支持多线程，而这七个资源管理器都支持调用Windows 的标准动态链库功能，所以只针对它们自身带有复制和移动功能进行测试（Turbo Browser 98和Explor2000 只支持 Windows 标准动态链库的功能调用）。测试结果表明：Windows Commander 4.0 Beta 2、Drag and File 、FileProbe和EF Commander都支持多线程功能，而Win Navigator不支持多线程功能（如图5）。

　　
　　　　　　　　　　图5

　　文件的查找功能也是资源管理器的主要功能。由于Explore2000和EF Commander本身没有内置查找功能，而使用Windows 的内核kernel32调用，所以它们查找速度与Windows 资源管理器一样，其余五个资源管理器都内置了查找功能。针对这些各具特色的查找功能，我们准备了包含490个目录（含多层目录O）共4712个文件来进行测试。为了得到更为准确的数据资源，这次也分别进行了10次测试，对其进行综合评比。根据我的测试结果，发现Windows 内置的查找功能相当强大，在我们指定的目录（包含4712个文件，490个目录）下查找所有文件时，Windows内置查找功能只用了2秒的时间，而采用内置查找程序的FileProbe的查找时间达23秒， Drag and File更甚，需32秒才能搜索完毕，不过，Windows Commander 4.0的查找功能相对其它内置查找程序的资源管理器（如图6），是最强大的，它搜索完指定文件仅用了5秒。由此看来，这些资源管理器程序在文件搜索上并没有获得Windows的核心技术。

　　
　　　　　　　　　　图6

　　另外，DOS命令虽然没有Windows 操作那样简洁、友好，但它操作比较快速、方便，对许多习惯于使用DOS命令的朋友非常有用，因为目前还有许多朋友在使用DOS命令。针对这些特殊用户，有些资源管理器专门做了一个DOS命令行的模块，如：Windows Commander 4.0、Win Navigator及EF Commander。

资源管理器特殊功能

　　我们在对资源管理器的基本功能评测的同时，也对它们所带有特殊功能进行了测试（包括文件分割、合并、压缩、解压缩，文件比较和目录同步的多种功能）。Windows Commander 4.0 Beta 2具有文件分割功能，表现在将一个大的文件按活动驱动器的大小自动设置为1.44MB、1.2MB、720KB、360KB和100MB（主要针对ZIP压缩文件）等六种方式进行分割。另外还可以由用户自行指定大小压缩（如图7）。并可以对分割后的文件进行合并。

　　
　　　　　　　　　　图7

　　为了测试这七个资源管理器对压缩文件的支持（压缩和解压缩），我们特意准备了10种格式的压缩文件，测试结果表明：Windows Commander 4和EF Commander都支持达9种，并且，Windows Commander 4.0和Turbo Browser 98还内置了ZIP压缩程序，在Windows未安装WinZip 或其它ZIP压缩程序的情况下能够压缩文件，也能正常解压，这一功能对于经常上网下载软件的朋友非常有用。这些支持解压缩功能的资源管理器，几乎都将压缩文件作为一个目录来管理，与目录具有相同的操作，包括浏览压缩文件中的所有文件及目录。

预览功能

　　对于文件预览功能而言，这七个资源管理器都能够对文本文件及常用图形（如：BMP、JPG、GIF、PNG、PCX、TIF、WMF等等）提供支持。但这些普通的预览功能根本不能满足用户的需要，现在有许多资源管理器提供了非常强大的预览功能。
　　Explor2000作为一个无菜单的资源管理器，它自身带有一个对文本文件、图形文件进行预览的功能，并且可以对文件进行16进制的编辑。Explore 2000可以直接浏览图形文件（与ACDSee看图软件一样），支持多达八种图形格式：BMP、JPG、GIF、PNG、PCX、TIF、WMF和ICO，可以直接在资源管理器中查看图形文件，不再借助第三方软件（如图10）。

　　
　　　　　　　　　　图10

　　对于资源管理器Turbo Browser 98可以让你直接浏览文件资料。其离线 HTML 过滤搜寻功能，可将你浏览过的网站并储存在cache目录里的所有 HTML 文件找出让你浏览。如果你认为资料对你有用，则可以另外储存于其他目录，反之则可立即删除它以节省你的硬盘空间。对于一些未安装Office的使用者，Turbo Browser 98可以直接让您浏览Office 的文件（包括 Word 97、Excel 等等）。当然，对于图形文件及多媒体文件，Turbo Browser98也都能立即开启浏览。

　　我们的测试结果表明：Turbo Browser 98无论是在文件预览功能，还是图形浏览功能都非常强大，应该排在这七个资源管理器之首。

资源管理器的FTP功能

　　现在，随着Internet的普及，我们经常要进行文件的上传和下载，而FTP（文件传输协议）就是这一功能的工具。制作一个漂亮的主页，已成为网民的愿望，用FrontPage 98等网页制作工具可以轻松做出，并且免费提供主页空间的网站也不少，多达几十兆，少则也有几兆。既然网页已经做出，主页空间也有了，就等着将这些网页传到主页空间。
　　所幸的是，我们的七个资源管理器就有四个支持FTP传输功能。
　　为了评测这四个资源管理器的连接FTP速度及从FTP上的下载速度，专门选在晚上进行（所有的同事都下班了，就可以独享128K的专线）。对四个资源管理器都使用同一台电脑，连接同一个FTP站点（ftp.microsoft.com）。由于存在时间上的差异，站点有时快，有时慢，可能造成对有的资源管理器不公平。同样，我们还是对每个资源轮回进行10次连接及下载（选择一个只有7983个字节的小文件），然后分别求平均值。Probe 1.01.01除一般的FTP连接外，还支持Sever 与 Sever 对传功能。
　　资源管理器Probe 1.01.01的FTP连接速度及下载速度都是最高的。虽然Windows Commander 4.0 的FTP连接速度最低，但它的下载速度仅次于FileProbe（如图11是Windows Commander 4.0 的下载窗口）。

评测总结

　　通过这次对七个资源管理器的测试，我们发现：作为一个资源管理器，它不仅仅是文件管理器，对文件进行最基本的简单操作（执行、复制、移动及删除），还具有文件分割和合并、文件压缩和解压缩、文件预览、图形浏览、FTP功能，以及播放MP3、CD或其它音乐文件等多媒体功能，而对整个系统的资源进行管理。
　　在评测当初，我们主要是希望通过测试，为用户选择一款适合自己的资源管理器，以脱离Windows资源管理器的束缚。然而，通过这次对多个资源管理器的评测后，我们发现这些共享软件与传统的Windows资源管理器相比，在功能上远远超过后者，是Windows资源管理器所无法比拟的。但这些资源管理器又是各具特色，互为补充，弥补别的资源管理器的不足。于是，我们便萌发出一个想法及希望：作为一个标准的资源管理器应具有的特色及功能，必须有友好的界面、与Windows 具有相似的操作风格、能够满足多种用户的需要、各种操作的速度比较快、资源的占用率比较低、支持文件预览和图形浏览、支持文件比较、文件分割及合并、压缩及解压缩、具有多种媒体（声音及图像）的播放功能，支持FTP文件传输功能。
　　本来希望在参选的资源管理器中能出现国产软件的踪影，毕竟我们都百分之百的支持和拥护国产软件，但对资源管理器进行收集时却无法找到一个国产资源管理器。所以，我们只好说：对不起，国产软件。我们希望中国的共享软件作者能更加努力，以中国人的勤劳和智慧，发展民族产业，让软件业与我们的国家和民族一起茁壮成长，屹立于东方，领先于世界，这正是我们的目标、我们的未来，同时，这也是一个市场、一个方向。
　　结合这次评测的结果，我们特选出几款具有特定优势的资源管理器。
　　1.本次最佳资源管理器：Windows Commander 4.0
　　Windows Commander 4.0虽然目前还只有Beta 2版本，但它在可操作性、基本操作功能（复制、移动、更名及删除等）都处于较好的地位，并且还具有ZIP压缩及多种格式的解压缩、文件分割及合并等功能，能够查看任何大小的十六进制、二进制及文本文件。另外，Windows Commander 4.0还支持中文（由电脑报编辑部的高大师汉化，我们从而获得一套Windows Commander 4.0的内部测试版），这又是它的一大优势。所以，我们将Windows Commander 4.0作为本次最佳资源管理器。据悉，在Windows Commander 4.0正式版中还将加入多线程的删除功能。
　　2.本次最具潜力的资源管理器：FileProbe
　　在我们的评测中，FileProbe无论是复制、移动，还是删除的速度都是第一名，并且在FTP连接及传输方面也居于领先位置。缺省有10种操作界面供选择（包括单窗口及双窗口等），具有Windows 风格。并且还可以将当前窗口状态保存到文件中，随时可以调入该文件以回到需显示的状态。FileProbe将FTP功能也加入到目录树中，除一般的FTP连接外，还支持Sever 与 Sever 的对传功能。FileProbe的这些功能与我们对一个标准资源管理器的要求比较接近，只是在多媒体及文件预览的功能上较弱，所以我们将它列为最具潜力的资源器。
　　3.本次操作最简单的资源管理器：Explor2000
　　作为一个多功能的资源管理器，号称“无菜单的资源管理器”，界面简洁，具有拖放文件功能、左右两个视窗及工具条，可进行目录及文件对比、目录打印、文件过滤、文件查看、拷贝、搬移、重新命名、删除、建新目录、文件查找及Hexa编辑等等,内置ZIP解压缩功能,可直接处理解压ZIP压缩包,可设定不同的文件格式以不同的颜色显示，方便管理文件，对于初学者很容易上手，并且功能能够满足所需。
