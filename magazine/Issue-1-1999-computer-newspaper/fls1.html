Windows 2000完全手册
●立飞 封立鹏

　　一、Windows 2000产品分类

　　Windows 2000是微软公司操作系统家族的新延伸，它集Windows 98和Windows NT 4.0的诸多功能和优良特性为一身。主要包括以下四个产品：
　　①Windows 2000 Professional：这是Windows NT Workstation新版本的新名称，主要是为商务用户开发，供台式机和便携机使用，它的目标是取代Windows 9X成为新一代的标准办公平台。
　　②Windows 2000 Server：这个版本
　　以前的名称是Windows NT Server 5.0，是在Windows NT Server 4.0的基础上开发出来的，是为服务器开发的多用途操作系统，可为部门工作组或中小型企业用户提供文件和打印、应用软件、Web和通讯等各种服务，是一个性能更好、工作更加稳定、更容易管理的网络操作系统。其最重要的改进是“活动目录（Active Directory）”技术，这是一种全新的目录服务技术，可以存储用户、组、策略、计算机和域的信息，极大地改进了在Windows NT Server 4.0中管理的不足。
　　③Windows 2000 Advanced Server：这个版本以前的名称是Windows NT Server 5.0 Enterprise Edition，除了具有Windows 2000 Server的所有功能和特性，还有一些专门为大型企业级服务器所设计的特性，例如群集、负载平衡和对称多处理器（SMP）支持等。
　　④Windows 2000 Datacenter Server：这是Windows 2000系列全新的版本。它是专门为数据服务器优化的，也是微软公司提供的功能最为强大的服务器操作系统，支持16路对称多处理器系统以及高达64GB的物理内存。它与Windows 2000 Advanced Server一样将群集和负载平衡服务作为标准的特性。
　　对于个人用户而言，Windows 2000 Professional是最佳选择，所以我们在下面着重介绍这一版本的Windows 2000。

　　二、Windows 2000 Professional的安装

　　Windows 2000建议的系统配置如下：
　　CPU：166MHz Pentium或更高频率（目前只支持单个CPU）。
　　内存：32MB RAM（推荐64MB）。
　　硬盘：2GB硬盘驱动器和至少650MB自由的磁盘空间。
　　我们在测试中发现，如果你的硬盘剩余空间低于900MB，CPU档次低于PⅡ266，那么Windows 2000的安装、运行对你来说简直就是一场恶梦。好在它的安装过程就像Windows 98那样完全智能化，需要用户干预、选择的地方很少，而且对于一个已经熟练使用Windows 98的用户来说，Windows 2000安装基本不会出现任何问题。但是整个安装过程大约需要一个小时以上（测试机型为赛扬366、64M RAM、10G硬盘）。Windows 2000 Professional的安装基本步骤如下：
　　1.MSDOS下的安装过程
　　在这个过程中，首先你需要退出现在运行的操作系统，选择一种方式引导到MSDOS，当然这时必须保证你的光驱是已经正常驱动。然后放入Windows 2000的安装光盘，进入安装目录键入“winnt” （注意键入时没有引号）开始安装。安装程序会检测你是否加载了“Smartdrv（磁盘高速缓存程序）”，如果没有，它会提示你加载。由于安装程序的第一步需要把大约200MB的安装临时文件拷贝到你的C盘中，所以如果你的C盘空间不够的话，将功亏一篑。和以前的Windows安装程序相同，你可以用参数来设置临时目录的位置，把安装临时文件放到其它的分区中，具体的命令信息请在DOS提示符下键入“winnt /?”来查询帮助。当临时文件拷贝完毕后，电脑会重新启动。接下来安装程序会询问你需要安装在那个分区中，一般情况下，你应该把Windows 2000安装在主DOS分区，而不是一个逻辑分区。选择了分区后，你需要选择以什么方式格式化这个分区，可以选择FAT或NTFS。既然我们要感受Windows 2000，毫无疑问我们应该选择NTFS，但这种格式的分区在DOS及Windows 9X中无法访问。格式化完成后，我们需要对Windows 2000定义目录名，由于它本质上还是Windows NT，所以我们依然给它命名为winnt。这时，安装程序会复制文件到Windows 2000的目录中，这大概又需要近十分钟时间。复制完毕后，系统会重新启动。
　　2.图形界面下的安装
　　终于，我们见到了久违的Windows安装图形界面，看上去十分精美。一开始我们需要等待，安装程序会自动检测硬件设备，这可是一个漫长的过程，如果你的硬件设备很多而兼容性又不是太好的话，你等待的时间可能会超出你的想象。
　　接下来我们可以看到定制的内容。首先是语言选择，我们当然选择简体中文，然后选择相应的键盘和输入法。此外还有区域选择内容，无非就是时区、数字、符号、货币、日期、时间等等，这与安装Windows 95/98都差不多，所以就略去不讲了。
　　和原来的Windows版本不同，直到现在你才看到版权协议说明。我们只有接受协议，不然就无法体验Windows 2000了。
　　关键的地方到了！我们需要设置计算机名称和管理员密码。熟悉Windows NT的人都知道这一步的重要性，所以建议你最好用纸记下你所设置的用户名称和密码，如果你忘了密码将无论如何都进不了Windows 2000。
　　最后，我们可以定制组件。Windows 2000自带了各式各样的组件，大家可以根据自己的需要来选择，如果你需要Windows 2000来做网络主机，那么IIS(Internet服务器)是必须要的，这个过程也是非常缓慢的。
　　3.最后的设置
　　以上一切都完成以后，windows 2000安装程序开始最后的收尾工作：安装开始菜单、注册组件、保存设置、删除安装过程中的临时文件。
　　4.启动Windows 2000
　　重新启动机器后，进入用户认证界面。拿出你刚才记好的计算机名称和密码将其填入。通过认证后，你就可以看到期待已久的Windows 2000的桌面了！

　　三、Windows 2000的新特性

　　1.稳定性较Windows 9X有了极大提高
　　可以肯定地说：Windows 2000绝对要比以前任何版本的Windows 9X要稳定许多。不过众所周知，系统的稳定性不仅取决于操作系统本身，而且与配套的硬件、软件和外设都有着非常密切的关系，其中很大一部分原因就来自于各式各样质量参差不齐的软硬件，所以微软在Windows 2000中提供了一些新的功能来确保系统的稳定性：
　　（1）系统文件保护方式――改良的内核写保护方式。
　　它禁止第三方应用程序替换任何位于Windows 2000系统路径上的动态链接库文件。当安装应用程序时，系统只允许在它们自己的路径上安装它们自己的动态链接库文件，而不能覆盖Windows 2000的动态链接库文件。
　　（2）Windows 2000安装服务。
　　它能够自动地修复被破坏的应用程序并跟踪系统共享资源的使用，防止在删除一个应用程序时删除另一个应用程序所依靠的共享文件。
　　（3）硬件映射保护。
　　这一特性可以标记包含执行代码的内存页并保护起来，这样即使是操作系统也不能向这些内存页中写入内容，从而可以有效阻止某些内核模式软件（如驱动程序）或OS代码所导致的系统崩溃，因为这些程序经常会试图向那些不能写入的内存中写入数据。由于有了这个保护，在Windows 2000中每一个程序都在它自己的那部分内存中运行，如果某个程序停止运行并不会影响系统中其它程序的运行。
　　（4）驱动程序检验。
　　Windows 2000提供了强大的、管理员可配置的机制，它可以让系统向用户报告其内核模式驱动程序中的错误，同时在不稳定的驱动程序相互作用时启动防御机制，这一功能有效地防止了不成熟或存在BUG的硬件驱动程序所导致的系统错误。
　　（5）改良的内存管理。
　　在以前版本的Windows中，内存管理不善是一个比较严重的问题，系统崩溃经常与这一点有关，而在Windows 2000中，我们明显发现内存管理有了很大的提高，系统能够较好地调配内存，使其稳定性大幅度提高。
　　2.桌面更加简洁友好
　　（1）开始菜单采用了个性化设计。
　　每一个计算机用户使用过程中都会安装许多应用程序，所以我们经常会发现不少用户的开始菜单非常混乱，其中有些是经常用到的，而另一些却可能只是偶尔才会使用，Windows 2000的个性化菜单特性将不断地监视并显示最经常使用的菜单项，其它不常使用的项目会隐藏在双箭头下面，这样不但可以让开始菜单变得简洁，而且使用户访问最常使用的程序的过程变得更加快速，有助于提高工作效率。
　　（2）增强的“我的文档”。
　　在以前的系统中应用程序没有将文件保存到同样的位置，所以导致了用户在使用中出现文件存放位置不一、查找困难的问题。在Windows 2000中的“我的文档”针对这些问题进行了增强，现在除非某个程序明确要求保存在不同的文件夹中，Windows 2000都会将应用程序的保存路径定向为“我的文档”，使其管理功能得到较大改善，对于公司或企业内部的用户来说，这种文件管理方式更为方便、快速。
　　（3）网上邻居更适应网络化。
　　为了使用户能够在他们整个的网络组织中方便地查找信息和资源，Windows 2000的网上邻居得到了相应的增强，原来显示在此文件夹中的用户所在工作组或域的计算机被放入了“网络邻居”的“邻近的计算机”文件夹中，而在“网络邻居”中建立的指向网络资源的链接可以是本地的，也可以是Internet上的FTP站点或Web文件夹。
　　（4）改进的资源管理器。
　　从Windows 95到Windows 98，资源管理器的变化不尽令人满意，而在Windows 2000中，资源管理器有了不小的变化，并且结合了不少IE 5.0的特性，比如在工具栏中增加了三个按钮：“搜索”、“文件夹”、“历史”，它们更有利于用户方便、快速地找到所需要的文件。而且它们不仅可以显示本地硬盘中的内容，还可以显示“网络邻居”、Internet上的内容。在Windows 2000的资源管理器中用户还可以对工具栏进行自定义，把那些常用的操作按钮放置在显著位置，而不经常用的可以将其删除掉。此外，Windows 2000还新加入了“打开方式”功能，用户可以利用右键命令中的“打开方式”来选择想要使用的应用程序打开某个文件。
　　（5）对白式的提示令使用者更为方便。
　　在Windows 2000中增加了对白式的提示，这样非常有利于用户发现系统的使用方法和具有的新特性，比如：当“个性化”开始菜单第一次生效时，就会出现提示告诉你在哪里可以找到那些隐藏项目，对于初学者可以说是非常体贴的设计。
　　（6）手写输入板。
　　在Office 2000中，我们第一次认识了微软的手写输入板，并被它深深吸引。现在，Windows 2000的“微软拼音输入法”也提供了这一功能。
　　3.系统配置和维护更加轻松方便
　　（1）重新定义的“控制面板”。
　　我们就选择几个具有代表意义或是较以前版本变化大的项目介绍一下：
　　添加/删除硬件
　　不仅是界面上的变化，也是功能上的增强，并且具有了诊断和卸载的功能，可以帮助用户添加、删除、卸载和调试硬件。
　　添加/删除程序
　　这个选项为使用者提供了更多有关应用程序的信息，比如所使用的磁盘空间、版本、使用频率和是否使用在线技术支持等。对于某些应用程序还包括了一个“修复”选项，它可以重新安装该程序来解决出现的问题。用户利用改进的“添加/删除程序”可以全面地了解系统中所安装的软件并且在一定程度上可以解决程序运行中的问题。
　　管理工具
　　在这里提供给用户尤其是系统管理员经常需要使用的管理部件，如计算机管理、组件服务、数据源、性能、事件查看器。
　　（2）强大的Windows安装服务。
这是Windows 2000中的一个新服务，可以帮助用户解决软件安装和重新安装中出现的问题，它为应用程序的安装定义并实施了一个标准格式，包括安装、修复、卸载、更新和组件的相关跟踪等。它可以使程序的运行更加可靠，同时减少DLL冲突。
　　（3）系统管理员的好助手――管理控制台。
　　微软采用了“Microsoft Management Console（MMC）来控制系统中所有的组件，这使得系统控制更加容易。微软对MMC做了很好的工作使它能够完全地控制计算机系统，而只用一个控制台便可管理所有的工具，你可以添加用户的帐户或为硬盘添加分区而不必打开或关上两个不同的程序。你还可以在设备管理器中查看硬件设备的故障等等。设备管理器和Windows 9X中的基本一致，它会显示出什么设备已经安装，占用了哪些中断地址，大大方便了用户阅读和查找设备信息。
　　（4）更多的启动选择。
　　当你在系统引导显示启动菜单时按下F8后，你将会看到七个启动选项：
　　・安全模式：和以前的Window 9X的安全模式一样。
　　・带网络连接的安全模式：除了与安全模式一样加载所需要的设备外，还加载启动网络所需要的服务和驱动程序。
　　・带命令行提示的安全模式：除了运行命令提示符而不是运行Windows资源管理器之外，与安全模式相同。
　　・启用启动日志：创建一正在加载的设备和服务的引导日志。
　　・启用VGA模式：只使用VGA驱动程序启动系统。
　　・最后一次正确的配置：让用户用最好的配置来引导系统。
　　・调试模式：通过串行电缆向另一台计算机发送调试信息。
　　（5）系统修复控制台。
　　“系统修复控制台”是Windows 2000的恢复和修理工具，可以让用户对损坏的或不能启动的系统进行修复和访问。
　　4.出色的多语言支持
　　Windows 2000的多语言技术允许用户使用60多种语言查看、编辑和打印信息，例如在英文版的Windows 2000中可以直接处理中文文档，而且对于我们使用简体中文的用户，以前查看繁体中文需要外挂平台，而现在不用安装任何其它的支持软件就能够方便地查看和编辑繁体的文档和网页了。
　　Windows 2000提供了三种语言版本：
　　英文版：可以使用多种语言输入、浏览和打印。
　　本地化语言版：有20多种语言的本地化版本，具有本地化的用户界面，可以使用多种语言输入、浏览和打印。
　　多语言版：能够切换使用多语言的用户界面，可以使用多种语言输入、浏览和打印。　　5.Windows 2000同样是移动用户的最佳操作平台
　　（1）新的脱机资源功能。
　　当计算机脱机工作时，文件和文件夹仍然像联机时一样处在相同的路径内，用户可以用联机时同样的操作访问网络文件夹和文件，当重新联机后，脱机时发生的更改将自动被同步化。
　　（2）新的高级电源管理技术。
　　Windows 2000支持ACPI，利用它Windows 2000能够管理系统的电源状态，以响应由用户、应用程序及设备驱动的输入。
　　（3）为笔记本专门设计的网络连接向导。
　　Windows 2000中的网络连接向导提供了五种类型的网络配置向导：拨号到专用网络、拨号到Internet、通用Internet连接到专用网络、接受传入连接、直接连接到另一台计算机。用户只需要根据不同的情况选择不同的配置向导即可创建多种连接，连接设置也是自动的，避免了下载和安装附加服务的麻烦。
　　6.其它新特性
　　・集成了DirectX 6.0。
　　・支持新的OpenGL 1.2规范。
　　・支持先进的TAPI 3.0技术。
　　・支持异步传输模式（ATM）。
　　・支持欧元货币符号。
　　・支持UNIX的互操作。
　　・全新的CD播放器，外观非常像汽车音响，通过播放器上的Internet按钮还可添加交互式的Internet链接，使用户可以得到更多的相关信息。

　　四、Windows 2000使用指南

　　(一)Windows 2000的桌面
　　Windows中的桌面即是我们工作的平台，常见的图标有以下几个：
　　“我的电脑”：管理本地计算机资源，进行磁盘、文件夹操作，可以对他们进行格式化、移动、复制、删除和改名；配置计算机的软硬件环境，例如配置打印机和拨号网络等。
　　“回收站”：暂存用户删除的文件、文件夹等内容，以便在需要时恢复。如果清除掉其中的内容，就不可能恢复它们了。
　　“网上邻居”：如果用户的计算机连接到了网络上，那么还会有“网上邻居”图标，通过它可管理网络中的资源。
　　“我的文档”：存放用户个人的文档。
　　“Internet Explorer”：用来浏览因特网信息。
　　很明显，和Windows 98/95相比，Windows 2000的图标变得更加好看，桌面的色彩变得更加明亮了，而且鼠标指针变成了带有阴影的样子，用鼠标进行单击菜单或者进行一些右键操作时，菜单的弹出都带有动画渐变效果，无疑这更加美观和吸引人。
　　1.任务栏
　　在桌面的下方是任务栏，任务栏包括子栏。这些子栏有“地址”栏、“链接”栏、“桌面”栏、“快速启动”栏和“应用程序”栏。此外，用户还可以根据需要添加自己的任务子栏。
　　添加新对象：可以从别的地方通过鼠标拖动的方法创建一个新的对象，当然也可以通过右键单击桌面来创建新对象。删除桌面上的对象：用右键单击桌面上的某对象，然后从弹出的快捷菜单中选取“删除”命令即可。
　　排列桌面上的图标对象：可以用鼠标把图标对象拖放到桌面上的任意地方；也可右键单击桌面，在弹出的快捷菜单中选取“排列图标”或“对齐图标”命令。
　　［注］：为使拖动图标操作有效，在“排列图标”级菜单中的“自动排列”选项必须被清除。
　　启动程序或窗口：只要单击桌面上相应的图标对象即可。
　　2.窗口
　　对于Windows系统而言，窗口操作是最基本的操作。什么是窗口呢？简单地说，窗口就是一个用户操作的客户区，它是一个完整的可运行的应用程序外壳，其中可以包含图标或文件的快捷方式，在窗口中可以进行各种各样的操作。在WindowS 2000中，不管是应用程序窗口还是文档窗口，其外观基本一致。我们可以对窗口进行的操作有：移动窗口和改变窗口尺寸；窗口的最大化、最小化、还原及关闭；窗口的切换（Alt+Esc或Alt+Tab键）；窗口的排列；拷贝窗口的内容。如果希望能把某个窗口的内容复制到另一些图像中去，可以按Alt+Print Screen键。
　　3.开始菜单
　　桌面上最重要的就是开始菜单。“开始”按钮总是位于任务栏的左端。打开开始菜单，移动鼠标，当指向菜单中带有三角形标记的菜单项时，就会打开相应的子菜单。（提示：用户还可以利用键盘打开“开始”菜单，为此只需按Ctrl+Esc即可。这一快捷键在任务栏处于隐藏状态的情况下特别有效。）
　　Windows 2000还允许将控制面板中的内容层叠，并作为开始的子菜单以便快捷地访问控制面板选项。你也可以改变开始菜单中的文档的设置，“我的文档（My Document）”文件夹也可以在开始菜单下级联排列。
　　在“我的电脑”或“Windows资源管理器”中启动应用程序：单击“开始”菜单中的“Windows资源管理器”图标，在弹出的窗口中选择应用程序所在的驱动器盘符，桌面上将弹出包含该盘上所有文件和文件夹的窗口，然后双击你要打开的程序即可。
　　在登录Windows 2000时启动程序：在Windows 2000的开始菜单中，有一个名为启动的文件夹。将一些应用程序的快捷方式放在这个启动文件夹中，就可以在Windows 2000登录后自动执行该应用程序（与Windows 95/98的启动项作用一样）。
　　4.Windows 2000的查找功能
　　Windows 2000具有更强的查找功能，这一点让我们感到欣慰。新的查找功能让我们提高了效率，它可以搜索本地以及网络上的驱动器，也可以搜索网络中的某台计算机。Windows 2000提供的查找功能有以下特点：
　　(1)新的HTML样式查找功能将代替以往Windows版本中开始菜单的查找项。Windows 2000允许你在任何打开的文件夹窗口中进行本地查找。而“在Internet上”会提供一个链接到流行搜索引擎站点的查找工具条，如AOL、Excite、Lycos和Yahoo，很象Windows 98的查找选项。除了保留原有的查找选项外，Windows 2000还添加了一些新的选项，如在高级选项中有大小写敏感等。如果你要查找某个人，系统将打开了一个单独的LDAP查找对话框，这和Internet Explorer 4.0和Windows 98中的一样。
　　(2)为了加快查找速度，你还可以采用Microsoft的索引服务功能，索引服务器把本地内容的检索作为后台服务，你可以在此确定哪些目录可以检索以及检索文件的基本属性。当然切换服务会导致性能上有所影响，但是影响是很小的，如果Windows 2000服务器附有Win2000 Pro系统，你可以让服务器停止内容索引的任务。在牺牲部分CPU和磁盘空间的同时，你将得到文件性能上的显著提升。首先最为明显的是查找速度加快了，例如在3,500个文件中按双字匹配查找920个记录只需要1秒钟。另外，在进行本机或网络的内容查找时，查找字符串之前如果使用了一个感叹号，则后面可用布尔运算符号，如“AND”、“OR”、“NOT”和“NEAR”。 Windows 2000的内容检索服务还可以提供特性查询，比如数字/比较符，以及带嵌套、输入条件加权和混合、多级的复杂查询。
　　5.Windows 2000的退出
　　在关机之前，像Windows 95/98一样，用户要确认退出Windows 2000，否则可能会破坏一些未保存的文件和正在运行的程序。如果未退出Windows 2000就强行关机，系统将认为这是非正常中断，在下次开机时会自动执行“Scandisk（磁盘扫描）”程序来检查系统文件并修正出现的错误以便使系统稳定运行。但尽管如此，非正常关机仍很有可能造成致命的错误并导致系统无法启动。所以我们建议用户在退出Windows 2000之前应关闭所有打开的程序和文档窗口。如果用户不关闭，系统也会询问用户是否要结束有关程序的运行。退出方法依然是采用“开始”菜单的“关闭系统”命令。单击“开始”菜单中的“关闭系统”选项或用“Alt+F4快捷键”，Windows 2000会提示用户三种选择（图1）：注销Administrator、关机、重新启动。

　　1.出色的工具栏
　　Windows 2000窗口的工具栏提供了更多的工具，并且可以由用户调整其显示方式。我们可以看到在“查看”→“工具栏”选项的右边有一个小三角，表示它有子菜单。将鼠标指针移到该菜单项上，即可打开下级子菜单。在子菜单中还有“标准按钮”、“地址栏”、“链接”、“电台”四个命令选项。其中“电台”这个命令选项是Windows 2000中新出现的选项。利用章四个选项，用户就可以自由地打开或关闭标准按钮工具栏及地址栏、链接工具栏、电台频道等。
　　“标准按钮(S)”
　　在这一栏中列出了部分菜单命令的快捷按钮，其中“搜索”、“文件夹”和“历史”按钮是以往Windows 9X系列没有的工具按钮，利用它们可以改变窗口中文件和文件夹的操作方式。这三个按钮功能比以前的Windows 9X系列按钮更加强大，其具体的使用方法我们将穿插在下面对按钮的使用介绍中。
　　“地址栏(A)”
　　地址工具栏是Windows 98中新增的，现在继续保留在Windows 2000中。在此栏的文本框中用户可以选择和输入一个文件夹名来改变当前窗口的显示位置和内容。在“地址栏”文本框中用户还可以输入一个网址，从而直接从这里进入网络访问。
　　“链接(L)”
　　在这一栏中列出了Microsoft公司推荐的网站，利用它可以十分方便地进入这些网站进行访问和下载所需要的资料。用户也可以在此栏中添加自己经常要登录的网站。
　　“电台(R)”
　　电台栏是Windows 2000中所新添的一个工具栏，由于Internet的技术突飞猛进，在网上收听音乐广播和视频广播已不再遥不可及，现在全世界已经有很多专门进行网络音频广播业务的公司，电台栏的出现就恰好满足了人们在网上收听音频广播的需要，用户只要点击电台工具栏上的小箭头，在弹出的下级菜单中选择电台指南，这时就会自动连上Internet找到提供相应服务的网站，然后添加到收藏夹中，只要设置没有问题的话，用户就可以在网上聆听实时网络广播了。
　　2.窗口的“缩略图”方式
　　这种显示方式只有在点击驱动器进入子目录的情况下才会出现，与以往不同的是用此方式显示当前文件夹和文件的时候，所显示的对象将变为一个个很大的方框，方框中的图标就表示当前对象是文件夹还是文件，文件夹还是以往的方式来表示，但是如果当前要显示的是图像文件的话，这时候方框中就会显示出这个图像文件的缩略图，这就好比一个小巧的图像观察器一样。缩略图显示尤其适用于当前目录中有许多图像文件。
　　(三)Microsoft管理控制台(MMC)
　　硬件设备的管理一直都是用户感觉最为棘手的问题，但在Windows 2000中，你会发现你的麻烦少了许多。如果你已经开始使用Windows 2000，就会发现它是一个性能极其稳定的操作系统，用户可以不必太多干预，就能将计算机维护得很好。
　　在以往Windows 9X下管理计算机的硬件资源和软件资源并不是一件很轻松的事情，而且必须要具有一定的计算机专业知识才行，但在Windows 2000中对于计算机的管理已经变成了一件非常轻松的事了。你可以利用“计算机管理控制台――MMC”来统一管理所有的硬件资源和软件资源，下面我们详细介绍一下MMC的使用。
　　1.管理控制台(MMC)
　　Windows 2000已经将大多数系统管理工具集成到了“Microsoft Management Console”（微软管理控制台）中，更易于管理，极大方便了用户的操作。它集成了以前Windows NT版本中的大多数管理工具，并将它们组织在一起。无论在用户界面还是功能上，都迈上了一个崭新的台阶。管理控制台（MMC）捆绑了若干Windows 2000管理工具到一个单独的控制台，提供了对特定计算机管理特性和工具的简单访问。它可以完成以下功能：
　　・监控系统事件如登录时间、应用程序错误等。
　　・生成和管理共享目录。
　　・观察连接至本地或远程计算机的用户情况。
　　・启动或停止系统服务如任务调度和打印缓冲。
　　・设置存储设备的属性。
　　・观察设备配置和添加新设备驱动程序。
　　・管理服务器应用和服务如域名系统或动态主机配置协议。
　　2.管理控制台（MMC）的使用方法
　　选择“开始”→“程序”→“管理工具”→“计算机管理控制台”。这是一台工作站上的本地的管理控制台(图3)。计算机管理控制台的界面类似Windows资源管理器的双窗口视图，左边的视图以树状分支方式列出各种系统工具，“导航和工具选择”包括管理工具、存储设备、服务器应用程序和本地或远程计算机上提供的服务。而树状控制台的三个节点分别是“系统工具”、“存储”和“服务器应用程序及服务”。

　　
　　　　　　　　　　　　　　　　图3

　　如果你选择了一个工具，右边的结果窗口中就会显示其相应属性、数据或可用的子工具，然后再利用右边窗口中的菜单和工具栏执行相关管理任务。
　　［注］：如果你希望使用计算机管理的所有功能，你必须是管理组的成员。否则，你将不能访问和修改管理属性，而且不能执行管理任务。
　　在计算机管理控制台窗口顶部包含两个菜单项：操作和查看，其中的选项是随着左边视图中所选工具的不同而变化的。在这两个菜单项后边是两个方向按钮，通过这两个按钮可以返回到已经查看过的系统工具。
　　(1)“系统工具”节点。
　　在缺省设置下，通过系统工具节点可以访问到六个系统工具：性能日志和警报、本地用户和用户组、系统信息及服务、共享文件夹、事件查看器、设备管理器。
　　(2)“存储”节点。
　　在缺省配置下，这个节点显示出计算机中安装的所有存储设备。通过此节点，你可以浏览和管理存储设备的属性，包括驱动器卷标、已用磁盘空间、访问权限和共享限制。你还可以在此节点下找到磁盘快照。
　　(3)“服务器应用程序和服务”节点。
　　服务器应用程序和服务是计算机管理树状控制台的第三个节点。通过这个节点，你可以浏览和管理安装在计算机上的任何服务器应用程序和服务如DNS或DHCP。这个节点包括以下的缺省服务应用程序和服务集（你实际看到的服务集可能有所不同）：Internet信息服务、电话服务、DNS、DHCP、Fax服务、RAS服务、路由服务、WINS。
　　3.系统信息
　　如果要查看系统信息，单击“系统工具”→“系统信息”使其展开，你会看到包含四个子节点，分别是：系统信息摘要、资源、组件、软件环境。单击系统信息摘要，这时右边的窗口将显示本机的系统信息列表。接着我们再展开资源节点，下一级显示出更详细的系统资源条目，其中很多资源条目在以前的Windows NT版本中只有在控制面板中才能见到。而在Windows 2000的计算机管理控制台中，这些条目被集成在一个节点下。
　　你也许已经注意到，在选择了“系统信息”后，计算机管理控制台顶部的菜单项中多了一项“工具”。单击展开它会看到九个选项：磁盘清理、Dr. Watson、硬件向导、网络连接、备份、文件签名验证应用程序、更新向导解除安装、WBEM权限编辑器、Windows报告工具。以下我们着重介绍以下选项的功能：
　　(1)“磁盘清理”工具。
　　与Windows 9X中的磁盘清理一样，我们已经很熟悉了。
　　(2)Dr. Watson。
　　它和Windows 9X/NT中的Dr. Watson相同，是一个系统检测程序，它会把系统及应用程序的错误写入一个日志文件中。
　　(3)文件签名验证应用程序。
　　单击“文件签名验证应用程序”选项，弹出“文件签名验证应用程序”向导，为保证Windows 2000系统的完整性，该工具将所有重要的系统文件进行数字标识，以便在它们更改时能被很快地检测到，单击开始命令按钮，签名验证工具开始检查硬盘上还有哪些系统文件没有被标识。单击高级按钮，在弹出的对话框中有两个选项卡：查找和正在记录。
　　①查找：
　　如果你只希望检查系统的完整性并将没有标识的系统文件通知自己，可以选择“如果有未经过签名的系统文件，请通知我”单选按钮。如果选择了第二个单选按钮“查找其它未经过数字标识的文件”可以指定需要查找的文件类型，单击下面的文件类型下拉列表框，从中选择文件的扩展名。然后在下面的文件夹文本框中输入要查找的文件夹名称，也可单击“Browse（浏览）”按钮来指定一个文件夹。
　　②正在记录：
　　单击“正在记录”选项卡，如果你希望将前面查找的结果保存到一个日志文件以便以后查看，选中“将文件签名验证结果保存到一个日志文件”复选框，该复选框下面还有两个单选框按钮：
　　附加到现有日志文件：将查找结果自动附加到上一个日志文件的后面，该日志文件的大小是随着查找次数的增加而增大。
　　改写现有日志文件：将查找结果保存为一个新文件并将原日志文件覆盖。
　　在最下面的文件名文本框中指定该日志文件的名称。单击“查看日志”还可以打开这个日志文件进行查看分析。
　　4.共享文件夹
　　使用共享文件夹可以显示本地和远程计算机的资源，在Windows 2000中，共享文件夹代替了Windows NT 4.0服务器版中的“控制面板”的一些相关部件。共享文件夹提供了所有共享资源的信息，在Windows 2000专业版中只有“Administrator”成员或者“Power Users”组成员可以使用共享文件夹。在Windows 2000服务器版中，“Server Operators”组的成员也可以使用共享文件夹。
　　（1）使用共享资源。
　　显示共享资源列表和打开文件：启动计算机管理控制台，在控制台“共享文件夹”系统工具下选择共享、会话或者打开文件。
　　断开用户连接：打开计算机管理控制台选择“会话”子项右击用户名，选择“关闭会话”，即断开该用户的连接。要想断开所有用户的连接，单击控制台顶上“操作”菜单，选择“终止全部的会话连接”，这样就会断开所有的用户连接了。
　　［注］：断开可能会使某些共享资源的用户丢失数据。
　　关闭打开的文件和资源：打开计算机管理控制台，选择 “打开文件”选项，如果要断开一个已经打开的文件或资源，右击文件名选择“关闭文件”即可。如果要断开所有文件连接资源，选择“操作”菜单，单击“终止全部的会话连接”。
　　［注］：断开可能会使某些共享资源的用户丢失数据。
　　（2）建立共享文件夹。
　　打开计算机管理控制台，选择“共享”选项，选择“操作”菜单，单击“新文件共享”子菜单，根据屏幕提示选择要共享的文件夹名，或创建新的文件夹名称。并根据提示设置好权限。单击可完成共享文件夹的路径和权限设置。
　　（3）停止文件夹共享。
　　打开计算机管理控制台，选择“共享”选项，用右键点击需要停止共享的文件夹，选择“停止会话”。
　　（4）改变共享文件夹的权限。
　　打开计算机管理控制台，选择“共享”选项，用右键点击需要设置的共享文件夹，选择“属性”中的“共享权限”选项卡，这里可以单击需要添加或更改的用户或是用户组名，另外在“权限”中，可以通过单击“允许”或者“拒绝”来设置本共享目录的权限。
　　（5）限制共享文件夹用户数量。
　　打开计算机管理控制台，选择“共享”选项，用右键点击需要限制用户数量的共享文件夹，在“属性”中的“用户限制”单击“最多用户”或者“允许”，若选择“允许”，需要在“用户”中指定数目。
　　（6）从共享文件夹中删除用户或用户组。
　　打开计算机管理控制台，选择“共享”选项，用右键点击需要删除用户和组的文件夹，选择“属性”，选择“共享权限”选项卡，点选要删除的用户和组。然后选择“删除”即可。
　　5.事件查看器
　　事件查看器可以说是一个老字号的管理工具了，早在Windows NT 3.51中我们就见过它，我们可以通过事件查看器轻松地了解当前系统所有的日志信息，迅速查看系统的一些非正常信息，从而排除所有的系统故障。在Windows 2000中这个经典工具的功能又得到了很大程度的加强，一般事件查看器有三种类型的事件日志：应用程序日志（包括程序记录的事件日志），例如数据库程序可以在日志中记载一个文件错误，程序开发者决定要监视的事件；系统日志，包括Windows系统部件记录的事件，例如驱动程序故障或启动的其它系统部件等；安全日志，包括合法的和非法的登录尝试及与使用资源有关的事件，如生成、打开或删除文件和其它对象，例如你打开了登录和退出登录的审计项目，则试图登录系统的事件就会记载在安全日志中。
　　［注］：事件日志服务一般在Windows启动时就会自动加载。不是所有用户都可以查看应用程序日志和系统日志，而只有管理员才能查看安全日志，在缺省情况下，安全日志的记录是未打开的。如果要打开它，需要运行域用户管理器，然后设置审计策略。管理员还可以在注册表中设置审计策略使安全日志写满时系统自动挂起。系统日志的查看和意义因为比较复杂，请参阅其它Windows NT/Windows 2000资料。
　　6.设备管理器
　　Windows 98中的设备管理器是系统属性中的一个部分，现在也搬到计算机管理控制台中了，同时功能更增强了不少。利用设备管理，你可以测试计算机上的硬件设备工作是否正常、改变硬件配置、识别为每个设备加载的设备驱动程序并获取设备驱动程序的信息、改变设备的高级设置和属性、安装升级的设备驱动程序、禁用和反安装设备、确定设备冲突并手工配置资源设置、打印计算机上安装的设备概要等。
　　在大部分的时间里你可以利用设备管理器检查你的硬件状态或升级硬件驱动程序，对计算机硬件熟悉的高级用户更可以利用设备管理器的诊断功能解决设备冲突和改变资源设置。不过在通常情况下，你不需要用设备管理器改变资源设置，因为Windows 2000在安装时已经自动分配了资源。设备管理器只能管理本地计算机的设备，对远程计算机而言，设备管理器只能工作在只读模式下。
　　（1）获取设备信息。
　　对于一个刚刚安装好的计算机系统而言，用户要如何才能获取设备信息呢？其实只要打开了计算机管理工具就可以清楚地了解到本地计算机的设备和资源设置了。方法为：打开“计算机管理控制台” → “系统工具”→“设备管理器”。以下的操作均以设备管理器为起点：
　　（2）浏览设备状态。
　　（3）浏览通用串行总线(USB)控制器的带宽分配。于每个USB控制器都有一定容量的带宽，所有连接的设备都必须共享，所以每种设备都有被消耗的带宽。查看被消耗的带宽可通过以下步骤进行：在“详细资料”窗口中，双击“通用串行总线控制器”，用右键点击“Intel PCI到USB通用主机控制器”，选择“属性”，在“高级”标签中，可以浏览带宽消耗设备列表中每个设备的带宽消耗。
　　（4）禁用一个设备。
　　在计算机的使用中有时候需要对某个设备进行禁用，禁用一个设备的方法如下：在“详细资料”窗口中，双击你想禁用的设备类型，用右键点击该设备，选中禁用。
　　（5）改变设备的资源设置。
　　在计算机的管理过程中，有时不得不更改设备的资源设置，比如某一个设备必须使用特定的中断和IRQ，但是这个资源又被另外一个设备占用，这时候就要考虑改变设备的资源设置，使之互相协调，共同发挥作用。
　　（6）改变端口设置。
　　在“详细资料”窗口中，双击端口“COM & LPT”，用右键点击需要改变设置的端口，选择“属性”，在“端口设置”标签中，改变相关设置。
　　［注］：大部分占用端口的Windows 2000设备自己会设定端口速率。调制解调器的缺省端口速率在控制面板中的电话和调制解调选项中设定。
　　（7）改变DVD驱动器的高级设置。
　　在“详细资料”窗口中，双击“DVD/CD－ROM驱动器”，用右键点击需要改变高级设置的DVD驱动器，选择“属性”，在“高级设置”标签中，改变相关设置。
　　（8）改变设备的能源管理设置。
　　在“详细资料”窗口中，双击改变能源管理设置的设备，右键点击该设备，选择“属性”，在“能源管理”标签中，改变相关设置。
　　（9）配置多媒体设备。
　　在“详细资料”窗口中，双击“声音、视频和游戏控制器”，用右键点击需要配置的多媒体设备，选择“属性”，在“属性”标签中，找到设备类型然后进行修改。
　　（10）卸载和重安装设备。
　　在“详细资料”窗口中，双击想卸载的设备类型，用右键点击该设备，选中“卸载”并确认。
　　（11）重安装即插即用设备。
　　在“详细资料”窗口中，双击想重安装的设备类型，用右键点击该设备，选中“卸载”并确认。然后在菜单中选中“扫描硬件变化”，遵循“发现新硬件”对话框的指导，如果发现新硬件对话框没有出现，很可能该设备不是即插即用设备。这种情况下，可以到控制面板中使用“添加/删除硬件”。
　　（12）升级和浏览设备驱动程序。
　　在“详细资料”窗口中，双击想升级或改变的设备类型，用右键点击“属性”，在“驱动程序”标签中，点击“升级驱动程序”打开升级设备驱动向导，然后遵循向导的指导。
　　计算机系统管理器还有几个重要的模块：用户管理、储存设备管理、Internet服务器应用程序和和服务等。由于篇幅所限请查阅相关的书籍。

　　五、Windows 2000网络基础

　　Windows 2000的网络系统由客户、适配器、协议和服务四大组件构成，它对现有的硬件设备、网络协议、网络服务系统有更好的支持，还可以提供多种服务。
　　1.利用Windows 2000建立对等型网络
　　对等型网络可以方便的实现资源的共享与信息的传递，特别适合组建小型或微型网络。Windows 2000内置了对等型网络功能，利用Windows 2000可方便的组建对等型网络。
　　（1）安装网络适配器。
　　无论用户想组建何种网络，在Windows 2000中安装适配器的方法都是相同的。在这里我们是以Dlink 530 TX 100M网卡来作为讲解的例子。具体的安装过程由于篇幅所限不再详述。
　　（2）设置识别数据。
　　Windows 2000利用识别数据来区分网络上的计算机。识别数据包括计算机名、工作组及隶属域三项内容。利用下面的方法可以对计算机的识别数据进行设置：
　　第一步：在“开始”菜单中，选择“设置”→“控制面板”选项；
　　第二步：在“控制面板”窗口中双击“系统”图标，打开“系统”对话框；
　　第三步：单击对话框中的“网络标识”标签，切换到“网络标识”选项卡；
　　第四步：在“网络标识”选项卡中，显示出当前的Windows 2000系统安装时缺省用来在网络上标识该计算机的计算机名称和所在的工作组的名称，点击“高级”按钮；
　　第五步：在弹出的标识更改对话框中，输入用户为计算机定义的新名称、用户希望加入的工作组以及想隶属于的域的名称。
　　［注］：在同一工作组中的计算机不能同名，否则网络将无法正确识别计算机。如果网络中的计算机较多则可将计算机合理地分为几组，使用户能更加方便的访问其它计算机。
　　（3）安装设置协议。
　　利用Windows 2000建立对等网络时，一般情况下可使用TCP/IP协议。安装设置该协议的方法如下：
　　第一步：在“控制面板”窗口双击“网络和拨号连接”图标，打开 “网络和拨号连接”对话框；
　　第二步：在“本地连接”图标上单击选择“属性”，打开如“本地连接属性”对话框。点击“安装”按钮；
　　第三步：在“请选择网络组件类型”列表框中双击“协议”选项，打开“选择网络协议”对话框；
　　第四步：在该对话框的“网络协议”列表框中单击“Internet协议（TCP/IP）”选项，单击“确定”按钮。
　　［注］：在网络上的计算机都要选择相同的通讯协议，否则计算机之间将无法传递信息。另外由于Windows 2000还支持其它多种协议，因此用户可以在同一台计算机中添加多种通讯协议，Windows 2000会自动选择一种适当的协议与其它计算机进行信息传递，在这里我们只选取了TCP/IP协议作为例子。
　　（4）安装设置客户。
　　在Window 2000对等型网络中，应使用微软网络客户，即“Microsoft Networks客户端”。添加客户的方法如下：
　　第一步：在“控制面板”窗口中单击“网络和拨号连接”图标，打开 “网络和拨号连接”对话框；
　　第二步：在“本地连接”图标上单击选择“属性”，打开“本地连接属性”对话框，点击“安装”按钮；
　　第三步：在“选择网络组件类型”列表框中双击“客户”选项；
　　第四步：在“客户”对话框列表中选择“Microsoft Networks客户端”选项，然后单击“确定”按钮。
　　（5）安装设置服务。
　　在用户正确安装了网络适配器、协议和客户以后，就可以使用其它计算机中的共享资源了。但如果对等型网络的计算机都是共享他人的资源而不提供资源共享，那么在网络上实际上将无资源可共享。如果希望装有Windows 2000的计算机提供资源共享，则必须为计算机安装服务组件。为Window 2000安装服务组件的方法如下：
　　第一步：在“控制面板”窗口中单击“网络和拨号连接”图标，打开“网络和拨号连接”对话框；
　　第二步：在“本地连接”图标上单击选择“属性”，打开“本地连接属性”对话框，点击“安装”；
　　第三步：在“请选择安装的网络组件”列表框中选择“服务”选项，然后单击“添加”按钮，打开“选择网络服务”对话框；
　　第四步：在“选择网络服务”对话框中的列表框里选择“Microsoft Networks的文件与打印机服务”选项，单击“确定”按钮，“Microsoft Networks的文件与打印机服务”就被添加到了网络组件中。
　　（6）设置共享文件、文件夹及驱动器。
　　在安装了服务组件以后，用户就可以设定计算机资源为共享资源了。设置共享资源的方法很简单，无论是在对等型网络还是其它类型的网络模式，设置共享文件、文件夹和驱动器的方法都是类似于以往的Windows NT 4，设置共享文件、文件夹及驱动器的方法如下：
　　第一步：在桌面上，双击“我的电脑”，打开“我的电脑”窗口；
　　第二步：右击C驱动器的图标，在弹出的快捷菜单中单击“共享”选项，打开“属性”对话框；
　　第三步：在对话框中单击“共享”选项卡，即可以设定驱动器C为共享资源；
　　第四步：在这里，我们可以看到驱动器C的“共享名”文本框中已经有了“C$”字样，在备注文本框中注明的是“Default share”字样，这是由于Windows 2000是建构于NT技术的网络操作系统，在安装好的Windows 2000系统中，每个驱动器都是作为缺省的共享资源，但相对于网络邻居而言却不会看见这些缺省共享的驱动器资源，其实秘密就在每个驱动器共享名后面的“$”字样，每一个共享资源的共享名后带了这个符号之后，在网上邻居中就不会看见此共享资源，要想共享此驱动器给其它用户访问，就必须新建一个共享资源；
　　第五步：单击“新建共享”，在弹出的对话框里我们可以输入一个共享名及备注信息，然后单击“确定”按钮；
　　第六步：至此，网络上其它用户就可以访问这台计算机的驱动器C了。当然用户还可以设置访问该共享资源的用户数及用户的访问权限。系统缺省的用户数是允许“最多用户”访问，你还可以限制访问的用户数，方法是点击下面允许用户访问的数字框来决定用户的访问数；
　　第七步：设定了用户数之后我们还可以对访问用户的权限再加以设置，单击“权限”按钮，在共享权限属性页中，你可以添加新用户和组来增加对该资源的访问，点击添加就可以添加新的用户和组；
　　第八步：在权限区域我们通过权限范围来设置用户的访问权限，如点取“读取”权限的“允许”，而把“完全控制”和“更改”权限去掉，则该用户只能得到读取驱动器C的访问权力。
　　［注］：在共享权限属性页中，选中“读取”权限的“允许”单选框时，网络上其它用户就只能读取这台计算机的驱动器C中的内容；选中“完全控制”的“允许”单选框时，网络上其它用户将能够读取、创建、修改、删除这台计算机的驱动器上的文件。
　　2.Windows 2000与Windows NT Server 互连
　　Windows NT Server 是时下最为流行的网络操作服务系统之一，由于同为Microsoft公司的产品，因此Windows 2000与Windows NT连接起来非常方便。
　　前面在对等网的连接设置时已经讲到过TCP/IP协议的安装，其实在连接Windows NT Server时Windows 2000 的TCP/IP协议的安装也是一样的，这里就不再重复了。下面从配置TCP/IP协议开始介绍如何进行设置以使Windows 2000能登录到Windows NT Server 4.0上。
　　（1）安装设置协议。
　　TCP/IP通讯协议过去仅在大型网络中使用，现在Microsoft 公司在Windows NT和最新的Windows 2000 中都提供了强大的TCP/IP协议组件，因此用户使用TCP/IP就可以建立Windows 2000 -Windows NT Server 4.0网络。
　　［注］：Windows 2000利用NetBEUI通讯协议也可以与Windows NT Server 4.0进行通讯，且不需要特别的设置。NetBEUI通讯协议的安装与TCP/IP协议的安装是类似的。
　　下面介绍在Window 2000中如何设置TCP/IP协议。
　　第一步：在“控制面板”窗口中双击“网络和拨号连接”图标，打开“网络和拨号连接”窗口；
　　第二步：在“本地连接”上点击右键选择“属性”；
　　第三步：在弹出的“本地连接属性”对话框中选择“Internet协议（TCP/IP）”选项，然后单击“属性”按钮，这时屏幕上将弹出“Internet协议（TCP/IP）属性”对话框；
　　第四步：在“Internet协议（TCP/IP）属性”对话框中“IP地址”选项卡中有“自动获取IP地址”和“指定IP地址”两个单选按钮。当在Windows NT 服务器中启动了DHCP服务时，请选择“自动获取IP地址”单选按钮；若Windows NT 服务器不能自动分配IP地址，则请选择“指定IP地址”单选按钮，并在“IP地址”和“子网屏蔽”文本框中输入相应的IP地址和子网掩码地址；
　　第五步：这里我们以10.1.1.108来作为本机的内部IP地址，子网掩码我们填入255.255.255.0，在网关这一项填入Windows NT Server的IP地址，如10.1.1.130；
　　第六步：如果局域网中Windows NT Server是一个域名服务器的话（DNS服务器），我们也可以填入10.1.1.130作为主DNS服务器的IP地址；
　　第七步：单击“确定”即可。
　　［注］：如果在“TCP/IP属性”对话框里还想进行其它的设置，通过点击“高级”按钮，可以设置TCP/IP通讯协议的WINS服务“Windows Internet Name Server（Windows国际命名服务器）”、DNS“Domain Name Server（域名服务器）”配置等属性。这些属性的数值都要根据实际情况来设置，如果用户不能正确的设置这些选项，则Windows 2000依然可以用TCP/IP协议连通Windows NT 4.0服务器，并且可以共享服务器上的文件的打印机等资源。
　　（2）登录到Windows NT 服务器。
　　完成上面的设置操作后，我们还要设置本机的网络标识以便登陆到Windows NT Server域中。
　　第一步：在“控制面板”中选择“系统”；
　　第二步：在“网络标识”选项卡中点击更改按钮，这时会弹出网络标识向导对话框，点击下一步，再选择“本机是商业网络的一部分，用它连接到其它工作着的计算机”，点击下一步；
　　第三步：选择“公司使用带有域的网络”，点击下一步，这时系统会提示你需要以下一些信息(图4)，点击下一步；
　　第四步：在出现的对话框中，键入Windows 用户帐号信息和域信息及相关的密码即可。当用户再次启动Windows 2000或者在“开始”菜单中选择“关机―重新启动”选项时，系统就会在启动过程中自动向Windows NT服务器登录，这个过程是不可见的，但是本机已经登录在局域网中Windows NT Server域中了。
　　（3）在用户级访问控制下设置资源共享。
　　当用户使用用户级访问控制时，为设置资源的共享，就需要有一台服务器为Windows 2000提供用户数据库。此时可以分别为每一用户设定权限。下面以共享驱动器C下的“My Documents”目录为例，介绍如何在用户级访问控制下设置资源共享。
　　第一步：双击桌面“我的电脑”图标，打开“我的电脑”窗口；
　　第二步：在“我的电脑”窗口中，双击驱动器C图标，打开驱动器C窗口；
　　第三步：右击驱动器C窗口“My Documents”，在弹出的快捷菜单中单击“共享”选项，打开“My Documents属性”对话框；
　　第四步：在“共享”选项卡中，单击“共享为”单选按钮，在“共享名”文本框中输入资源的共享名，例如“MYDOC”；
　　第五步：在用户数限制区，选择最多用户还是设定用户数；
　　第六步：单击“权限”按钮，打开“共享权限”对话框，点击“添加”按钮，弹出的“选择用户、计算机或组”列表框中列出了由服务器提供的用户及用户组的列表；
　　第七步：在列表框中选中要赋予访问权力的用户或用户组，选择好之后按“确定”按钮；

　　第八步、如果用户希望赋予选中的用户或用户组以只读的访问权力，则应单击权限区域中“读取”权限中“允许”复选框；如果用户希望赋予选中的用户或用户组以完全访问的权力，则应单击“完全访问”权限的“允许”复选框；
　　第九步：单击“确定”按钮，完成共享设置。完成共享设置后的“My Documents”将会出现一个手托着文件夹的形状。
