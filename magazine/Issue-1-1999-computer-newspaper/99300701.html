　　为什么要重复启动才能正常工作
　　重庆读者王华问：
　　1.我的主板是华硕TX97－E。我的朋友最近给我一块K6－2 266，安装好之后，进入Windows 95的蓝天白云时出现花屏（颜色不对），而且运行程序时感觉系统性能降低。这是否由于TX97－E不支持K6－2 266引起？
　　2.电脑在第一次开机（冷启动）时绝对死机，按热启动键之后可正常启动。所以我只好在“Starting Windows 95”之前就启动两次。更换CPU之后也是如此，检查没有病毒，请问是什么原因？
　　答：第一个问题，华硕TX97－E主板手册说明能够支持的最高外频为66MHz，倍频为3.5，因此支持的CPU最高工作频率为233MHz。不知道你在更换CPU时，是否重新设置了跳线，如果跳线设置不正确，肯定会影响系统正常工作。当然，即使按照上述跳线设置，也没有达到CPU的标称工作频率，而是降频使用，但不会影响系统性能。此外Windows 95系统安装时对硬件有识别和“记忆”能力，在更换CPU之后最好重新安装Windows 95系统，否则有可能影响系统性能。
　　第二个问题，因为是在引导操作系统之前就出现死机，而且重复启动之后就能够完全正常工作，因此病毒的可能性较小。可能与主板和开关电源有关。
　　为保护微机的工作稳定可靠，开启电源后，微机电源的POWER GOOD信号线在5V和12V准备好后延时0.1－0.5ms向微机主板输送一个幅度为5V的微机自检启动信号，如果该信号不能及时升达5V均会导致系统不能正常启动。在POWER GOOD信号正常之后重新复位，才能正常启动。在业余条件下检查有一定困难，可更换开关电源试一试。
　　如果能够排除开关电源的问题，可进一步检查主机其它部分。有可能是某一设备的电解电容或钽电容漏电，需充电一定时间方能慢慢恢复，个别半导体芯片故障也有可能，只能逐一取下板卡试试，到只剩显示卡还未解决问题时，通电后用手触摸板上各电容、芯片看有无异常发热现象，如果电容或者芯片有异常发热现象应当更换。
　　CMOS参数不能修改怎么办
　　广州读者魏忌伟问：
　　一台586微机，福扬主板、K6－2 233CPU。开机检测内存、硬盘、CD－ROM之后出现一段英文提示“CMOS checksum error－Defaults loaded”和“CMOS battery failed”。而进入CMOS Setup修改参数后不能保存，重新启动故障依旧。请问应该如何处理？
　　答：从提示看，CMOS参数是被修改了，使得其校验和（checksum）错误，因此不能正常启动。原因可能是两个:一是病毒修改CMOS参数;二是CMOS电池或者电池电路故障所至。处理办法如下：
　　1.首先试试杀毒，在上述提示出现之后按F1，先进入Windows 95或者DOS，然后清除病毒；
　　2.检查CMOS电池和电池电路，在断开电源之后看看CMOS电池的输出电压是否正常，该电压能否加到CMOS所在的芯片上（不同的机器采用的芯片不同）。
　　3.请你仔细检查一下主板，看看是否有禁止写入CMOS的跳线或者开关（有的主板有此开关）。如果有这种跳线开关应该把它设置为允许写入，才能存入修改的CMOS参数。
　　4.如果上述办法解决不了问题，有可能是CMOS RAM本身出了问题，使得无法写入。如果是这样，那就很麻烦了（因为CMOS RAM已经集成到控制芯片中了，而在业余情况下更换芯片几乎是不可能的）。一般说来你只能更换主板了。
　　为什么硬盘会自动分为两个区
　　湖南衡阳读者黄钢问：
　　我的机器配置是：Pentium 166CPU、华硕T2P4HX主板、蓝点声卡、丽台S600显卡、火球1.2G硬盘（分区为C：，主硬盘）、Seagate硬盘（从盘，分为D：、F：），8速光驱。一次开机时检测不到硬盘，断开从盘后用软盘启动。对主硬盘高级格式化后，发现硬盘有两个分区C：、D：，在C区装DOS，D区自动也有同样的DOS，怀疑有病毒，用DM501对其低级格式化，且只分为一个C区，高级格式化后，同样发现硬盘存在C区和D区，在C区装DOS和光驱驱动程序，自动在D区也有相同的DOS和光驱驱动程序，从硬盘和主硬盘同样的现象，用KV300 检测不到病毒。请问这是什么原因，应该如何解决？请专家予以解答。
　　答：来信所说的问题病毒的可能性不大，很有可能是你的CMOS参数中的硬盘工作模式设置不对。你的主板采用EIDE接口，支持三种硬盘工作模式：NORMAL、LBA和LARGE模式。
　　・NORMAL Mode 普通模式
　　这是原有IDE方式。在此方式下对硬盘访问时,BIOS和IDE控制器对参数不作任何转换。在普通模式下支持的最大柱面数为1024,最大磁头数为16,最大扇区数为63,每扇区字节数为512。因此支持最大硬盘容量为：512×63×16×1024=528MB。即使硬盘的实际物理容量更大,但可访问的硬盘空间也只能是528MB。
　　・LBA (Logical Block Addressing)Mode 逻辑块地址模式
　　这种模式所管理的硬盘空间突破了528KB的瓶颈,可达8.4GB。在LBA模式下,设置的柱面、磁头、扇区等参数并不是实际硬盘的物理参数。在访问硬盘时,由IDE控制器把由柱面、磁头、扇区等参数确定的逻辑地址转换为实际硬盘的物理地址。在LBA模式下,可设置的最大磁头数为255,其余参数与普通模式相同。由此可计算出可访问的硬盘容量为：
　　 512×63×255×1024 = 8.4GB
　　・LARGE Mode大模式
　　当硬盘的柱面超过1024而又不为LBA支持时可采用此种模式。LARGE模式采取的方法是把柱面数除以2,把磁头数乘以2,即使柱面数减少,但总容量不变。例如,在NORMAL模式下柱面数为1220,磁头数为16,进入LARGE模式则柱面数为610,磁头数32。这样在DOS看来柱面数小于1024,即可正常工作。相反的转换进程由BIOS的INT 13H完成,以便取得正确的硬盘地址。LARGE模式支持的最大硬盘容量为:
　　521×63×32×512=1GB。
　　你的硬盘容量为1.2GB，从上面的知识可知，如果你采用了NORMAL Mode （普通模式），将无法管理整个硬盘，这种情况下可能出现C、D两个分区，但是又无法正常使用的现象。请你进入CMOS参数设置程序，把硬盘的工作模式设置为LBA即可（笔者遇到过相似的故障，其原因也在于此）。应该注意的是，低级格式化、分区、高级格式化都必须保证在LBA模式下进行。否则仍然会出现一些更为奇怪的故障现象。
　　如何判断软盘子系统故障
　　山东济南读者解宇舟问：
　　我的机器配置为：Cyrix 233 CPU，TW SLS5571主板，三星3.5英寸软驱。几天前，突然发现软驱始终亮灯，接线一切正常，并不能通过自检，借他人的软驱试验，发现自检无问题但软驱不读盘，系统提示“机器未准备好”，反复试验均无效。检查过BIOS一切均正常，怀疑是软驱接口（主板）坏了，但为何能通过自检呢？请专家指教！
　　答：可以分三个区域检查来确定故障点：软驱、连接电缆、软驱控制器（接口）。
　　1.更换软驱，你已经作了试验，同样不能正常工作，说明软驱不一定有问题；
　　2.检查软驱连接电缆，应该换一条电缆试一试。同时检查主板上的接口插座有无大量灰尘（造成局部短路，使不能通过自检或者不能读盘）；
　　3.在BIOS设置中，禁止使用主板上的软驱接口，另外加一块多功能卡，把软驱电缆连接在上面，如果能够正常工作，说明软驱控制接口确有故障。
