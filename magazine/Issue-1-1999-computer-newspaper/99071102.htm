9种典型职业的软件选配套――之银行职员篇

　　一、我是银行职员
　　在别人心目中，银行职员就是跟金钱打交道的人。对于银行职员而言，他们是和数字打交道的人。以前的老一辈同行所面对的是账本和算盘，而我到银行工作的这五六年时间，感受到自己正处在一个计算机普遍应用的数字化世界里。
　　目前，我工作所处的部门在具有完备计算机网络体系的银行中，属于支行一级。作为一名银行的系统管理员，在这里我向大家介绍我们工作中所使用到的计算机硬件和软件，以及如何使用它们来完成日常的工作。
　　二、我的常用软件
　　在我们的部门有很多PC单机和一些基于Windows操作系统的办公软件，比如使用Visual FoxPro编写的人事管理系统、用Excel实现的报表和固定资产汇总软件等。在Windows 98系统平台上，我有时也用微软的开发工具编写一些小程序用来简化自己的工作；也有用Internet Explorer浏览WWW、用CuteFTP和兄弟行交换数据文件、用Foxmail发送电子邮件的时候；也和很多贪玩的系统管理员一样，用WinDAC抓CD音轨、用MPLifier压制MP3；用Photoshop处理自己喜爱的图片，有时，还偷偷地打游戏！毕竟，我自己不是个单调得满眼只识数字的人。
　　三、我们这行用的软件
　　银行的网络结构体现了典型的大型分布式网络计算机的特点。其主要应用均基于传统的Unix主机/终端机模式，而不是当前流行的服务器/客户机结构。采用这种方案的原因，除了历史因素的影响外，主要是出于对银行业务系统安全性考虑的。
　　我所在银行的网络由三个层次组成，它们分别是所有基层营业网点、（县）支行和（市）分行。为保障数据安全，对数据实行了高度集中的管理办法。在支行一级，所有业务网点均通过使用拨号专线连接的远程Unix终端，访问支行服务器数据库；在分行一级，所有支行服务器均通过X.25专线与分行服务器连接，实现数据向分行数据库的实时集中；对于跨省、市的业务，均由本地服务器向远程服务器发出交易请求，由对方主机处理后，返回交易结果。
　　银行业务的计算机处理系统通常被分为储蓄和对公两个主要模块，在我所处的银行，这两个系统事实上是分别由两台不同的主机来完成的。储蓄系统的硬件环境由一台HP NetServer LD Pro服务器、一台IBM 路由器、一台四方MFU多路复用Modem池组成；软件环境由SCO Unix Open Server V3.0操作系统、Sybase V10.0 for Unix大型数据库和银行内部开发的储蓄专用软件组成。对公系统的硬件环境相对简单，硬件平台是一台IBM P/75的商用PC；软件由SCO Xenix V3.2.4、MF Cobol －2组成，应用软件也是使用银行内部开发的软件。
　　如果认为银行系统管理员比较容易胜任，那就是大错特错。首先，我们的系统采用的是Unix操作系统，这个强大的操作系统在管理上的复杂性，需要操作者具备比PC机的操作系统多得多的理论知识。管理经验更是维持系统正常运行的关键，事实上，我这个具有7年系统管理经验的系统管理员仍在不断学习、积累之中。此外，我们的应用系统使用大型数据库Sybase作为系统数据平台，要想实现对这个数据库的管理，就必须熟练地掌握和灵活应用SQL（Structured Query Language）数据库查询语言。这是一个非常强大的数据库管理工具，但它像FoxPro和Access那样简单、容易上手。不过，在数据出错的时候，Sybase那烦琐的安装过程和耗时的数据恢复操作实在令人头疼。
　　四、我的一天
　　8：30开始上班。到达机房，依次打开UPS、路由器、Modem池和主机，然后初始化数据库、运行业务维护程序、核对业务系统运行参数，接收分行的电子汇兑数据，并完成分发操作（银行汇款24小时到账的承诺全部由它实现）并打印昨日联行数据。
　　9：00～18：00，监视业务系统和网络的运行状态。在实际工作中，系统管理员的作用更多是在系统出现突发事故时，才能体现出来。当出现故障后，我们系统管理员必须承担起迅速排除故障的任务。
　　18：00，营业终了，发送电子汇兑数据到市分行，轧账、打印当日业务报表并备份当日数据。(四川 李恒)
