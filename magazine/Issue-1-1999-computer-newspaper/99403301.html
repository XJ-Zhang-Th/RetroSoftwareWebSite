通过Internet实现远距离教学

　　多媒体远距离教学的目的就是利用成熟的通信技术和计算机技术，在不同的区域建立统一的教学模式，把高质量的教学传播到尽可能远的地方。对学员来说，就像进入了一所虚拟学校。教师给学员授课、答疑、解决问题，但两者却相隔千里，甚至同时参加学习的同学可能从未见过面。
　　在这种采用Internet网进行远距离教学的方式中，教学总部根据各个教学点学员的需求，组织、设计、开发课件，统一装入课件服务器中，由各个教学点通过Internet网进行学习和访问。同时教学总部也编制部分实物教材，如教科书、CD、VCD、DVD、VHS等教学介质，通过教学点提供给学员，每门课都有老师在Internet网上负责解答问题，在教学点也有授课教师，学员可以到教学点上课，也可以通过计算机和Internet进行网上学习、网上答疑、网上讨论。最后由教学总部点在网上组织考试并颁发证书。
　　在这种远距离教学方式中，教学总部由教学中心和技术支持中心组成。教学中心的职责是负责从教学到考试的每一环节的具体实施，组织、安排学员进行培训和学习；技术支持中心负责课件的编写、开发和制作，编制实物教材，如教科书、CD等，以及整个教学系统及网络的管理和维护。在各地设立的分教学点，由教学总部进行统一管理和协调。
　　具体的教学方式采用把实物教材（教材、CD、VCD）发给每个学员，学生根据自己的实际情况，可以到各个分教学点上课学习，或者通过计算机互联网（Internet）进行网上学习、网上答疑、网上讨论、网上信息发布、网络课件下载、E－mail通讯等。在有条件的地方可以通过可视电话或Internet会议系统进行双向交互式的远距离课堂教学、讨论和答疑。

Internet远距离教学系统的网络结构

　　图一所示的，是这种远距离教学系统的网络结构图，它是利用Internet网把各个教学点的局域网连接起来。其中的一个教学点,如深圳, 是系统在Internet网上的技术支持中心，它包括Web服务器、域名服务器、E－mail服务器、代理服务器和课件服务器，是整个远距离教学系统的支持核心。各个分教学点，如广州、上海等，是以通信服务器为核心建立的一个教学局域网（LAN）。每个教学点的LAN和一些独立的远程终端，通过Internet网与教学总部的Web服务器进行连接，构成一个完整的远距离教学系统。该系统结构还可以通过Internet网与国外的教学网络进行连接,这样可以很方便的引进国外先进的多媒体教学课程,对国内学员进行培训。

技术支持中心的实现方式

　　图二是技术支持中心的系统结构图，它由Web服务器、E－mail服务器、域名服务器、代理服务器和课件服务器构成。代理服务器把Web服务器、E－mail服务器、域名服务器和课件服务器分成了内部和外部。代理服务器、Web服务器、E－mail服务器、域名服务器通过HUB与路由器相连，路由器通过同步MODEM，用数据专线DDN或“一线通”ISDN接入Internet网。内部课件服务器通过交换机Switch，把几十台教学终端连接起来，构成教学点的局域网络。Web服务器将是各个分散教学点教学的支持核心。新建立的分教学点，只需要构造教学点的局域网，Web服务器就可以通过Internet网对各个教学点进行教学支持。

远距离教学构件

　　1.服务器（Server）
　　服务器是该系统在Internet网上的支持中心，是由Web服务器、E－mail服务器、域名服务器、代理服务器、防火墙构成。Web服务器是支持整个远距离教学系统的核心，负责各个分教学点(如广州、上海等)以及远程终端的非实时交互，实现网上学习、网上答疑、网上讨论、网上信息发布、网络课件下载、E－mail通讯等。其软件平台由NT Server 4.0、SQL Server、Exchange等构成。代理服务器负责网络的安全，并管理客户端（Client）关于Internet的请求。
　　2.课件服务器(Classware Server)
　　课件服务器装载着支持教学的课件库，教学点中的每个终端都可以访问课件服务器的内容。课件服务器可使用目前较先进的VOD(视频点播系统，采用多媒体网络技术、多媒体数据库技术和多媒体压缩技术实现的点播系统)技术来实现。它可以把CD、VCD、VHS、DVD介质上的图形、图像、声音、文字等多媒体信息，通过该系统装载到VOD服务器上，并建立高速检索。学员可以任意选择所需要学习的课件，并可以删除或重排播放顺序，而且，每门课可以供20～60个学员同时选择。
　　3.网关（Gateway）
　　网关是把具有不同网络连接体系结构的两个计算机网连接起来的功能装置。它可以用路由器和节点机来实现，路由器将数据分组向另一个路由器递交，直到数据分组被传送到其最终目的地为止。我们建立在Internet网上Web服务器的网关，就是选用路由器（Router）来实现的。
　　4.防火墙(Firewall)
　　防火墙是在内部局域网（LAN）与Internet网之间设置的安全装置，允许局域网与外界通信，但对外界流入的信息则进行有选择的控制。它可以用硬件或软件来实现，在我们的系统中选用Microsoft proxy软件实现。
　　5.交换机(Switch)
　　用来建立局域网的连接设备。通过它把服务器、终端连结起来，成为一个局域网。
　　6.安全特性
　　由于系统是建立在Windows NT安全机制模型之上，Windows NT安全机制通过请求分配用户账号和密码帮助保护计算机及其资源。管理者可通过限制这些账号的用户权力来控制访问资源，也可以使用“Internet服务器”设置只读或只执行虚拟目录。Internet Information Server 还提供了一种方法，用特定的ID地址或域名来控制用户访问Web服务器的权限。
　　7.通信网络
　　通信网络的选择直接关系到系统运行的速度和费用。从系统结构上可以看出，教学点的终端需要通过Internet网同远距离的教师进行问题讨论和答疑，这样我们需要通过ISP（Internet接入服务提供商）提供的联机服务入网。
　　8.通信线路
　　从系统结构图上可以看出，我们建立的Web 服务器与数据通信局的Chinanet之间需要有一条通信线入网，根据信息流量的大小和所要求的速率，通信线路可以选用 “一线通”（N－ISDN）或专线(DDN)。
　　（1）“一线通”（N－ISDN）
　　目前新推出的“一线通”（N－ISDN）业务是在现有的普通电话网上实现一种集语音、数据和图像传输于一体的综合业务。采用ISDN的数字通道，加装ISDN适配卡或外置AT设备就可以接入Internet网。通过ISDN可以方便地进行异地局域网互联，若加装摄像头及相应软件（或直接用可视电话终端）即可实现点对点的视像通信。
　　（2）数字专线（DDN）
　　数字专线（DDN）是目前使用最多的一种方式，它速率高、误码率低、不会断线，但使用费用比较高。
　　9.域名申请
 　　作为Web服务器提供者在Internet必须有一个域名，它代表着Internet上不同网站的名字，是企业在Internet上供别人访问的地址。
　　10.主页制作
 　　是整个远距离教学系统的用户界面, 是用户进入和使用该系统的桥梁，系统主页的制作好坏将关系到用户能否方便、直观、高效地使用该系统。

特　点

　　这种通过Internet网进行远距离教学的方式，系统采用的技术都是目前比较成熟的计算机技术、网络技术和通信技术，同时还具备了其他传统的教学方式无法比拟的特点：
　　1.学员的学习方式具有很强的灵活性。如果学员自己具备上网的条件，他完全可以在家里或者单位上通过Internet进行学习并接受老师的指导。如果自己无法上网，学员可以使用教学点提供给学员的部分教材，如教科书、CD、VCD、DVD、VHS等自行学习，同时可以到教学点上课，或通过教学点的计算机进行远距离网上学习、网上答疑、网上讨论。
　　2.这种实现方式具有较好的扩充性，几乎不受地域的限制，只要通电话的地方都能够很方便地“克隆”分教学点。每个分教学点的入网计算机数量可根据学员的情况进行配置。
　　3.实现了资源最广泛的共享。参与教学的教师不仅仅局限于一所学校，一个地方，通过Internet网可以汇集全国乃至世界上优秀的老师来培训学员。
　　4.通过Internet网进行远距离教学具有及时性，Internet网是目前人类克服空间和时间的限制，以最快速度传递信息的最佳手段，用Internet网进行远距离教学，就能够提供给学员们当前最新的教学内容。
　　5.通过Internet网进行远距离教学可以充分体现交互性，尽量满足学员个体的要求。 学员通过网络能够方便地同老师交流，学员之间也可以方便地联系，学员的各种问题都能够尽快地得到解答，对同一个问题探索不同求解方案，这样对学员来说更具挑战性，更利于创新能力的培养。
